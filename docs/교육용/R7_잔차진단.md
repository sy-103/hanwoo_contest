# R7. 잔차 진단 — 내 회귀를 믿어도 되는지 검사하기

> **이 강의의 목표**: [R2](R2_선형회귀_기초.md)에서 회귀의 4가지 가정(선형·독립·등분산·정규)을 배웠죠. 이번엔 그 가정이 **실제로 지켜졌는지 검사하는 법**을 배웁니다. 잔차 산점도·Q-Q 플롯·등분산 점검·쿡의 거리로, "내 회귀 결과를 믿어도 되는가"를 눈으로 확인합니다. 실습은 21회차 강의안(`21_잔차_영향력진단.md`, 작성 예정)과 연결됩니다.
> **앞 강의**: [R2](R2_선형회귀_기초.md)의 4가정·잔차, [R6](R6_실습_코드따라하기.md)의 OLS 적합.


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 회귀 7/8**

## 1. 왜 잔차를 "진단"하나

[R2](R2_선형회귀_기초.md)에서 회귀 계수를 **믿으려면 4가지 가정**(선형성·독립성·등분산성·정규성)이 맞아야 한다고 했습니다. 그런데 모델을 돌리면 계수와 p값이 **일단 숫자로는 나옵니다** — 가정이 깨졌어도요. 문제는, 가정이 깨지면 그 숫자(특히 신뢰도)가 **틀린 값**이라는 겁니다.

그래서 결과를 보고하기 전에 **"이 회귀가 건강한가"를 건강검진**합니다. 검진 도구가 바로 **잔차(residual)** 입니다.

> **잔차 복습**: 잔차 = 실제값 − 예측값. "이 소를 얼마나 못 맞췄나"입니다([R2](R2_선형회귀_기초.md)). 잔차에는 모델이 **설명하지 못하고 남긴 것**이 다 담겨 있어서, 잔차를 들여다보면 모델의 문제가 드러납니다.

> 핵심 비유: 잔차는 회귀의 **"검진 결과지"** 입니다. 건강한 회귀의 잔차는 **아무 패턴도 없이 무작위로** 흩어져 있어야 합니다. 잔차에서 무슨 **패턴이 보이면**, 그건 "모델이 놓친 게 있다"는 신호입니다.

---

## 2. 잔차 산점도 — 가장 기본 검진

**가로축에 예측값, 세로축에 잔차**를 찍은 그림입니다. 가장 많은 정보를 주는 한 장입니다.

### 건강한 모습 (가정 OK)

```
잔차
 │   · ·  ·  · ·   ·
 │ ·  · · ·· · ·· ·    ← 0을 중심으로 위아래 고르게, 폭이 일정,
─┼─·──·──·──·──·──·──    아무 패턴 없이 무작위
 │ · ·· ·  · · ·  ·
 │  ·  · ·  ·  ··
 └────────────────── 예측값
```

0을 기준으로 **위아래 대칭, 폭이 일정, 무늬 없음** → 좋습니다.

### 병든 모습 ① — 곡선 무늬 (선형성 위반)

```
잔차
 │ ·              ·
 │  ··          ··     ← U자/역U자 곡선이 보임
─┼────··────··──────
 │      ····
```

잔차가 **휘어 있으면** 실제 관계가 직선이 아닌데 직선을 그은 겁니다. → **제곱항(비선형 항)** 을 추가해야 합니다([R8](R8_강건성과_총정리.md)의 사양 개선). 비유: 굽은 길에 자를 대고 직선을 그으면, 군데군데 자가 길에서 벗어나는 패턴이 남죠.

### 병든 모습 ② — 깔때기 모양 (등분산성 위반 = 이분산)

```
잔차
 │           ·  ·  ·
 │        ·  · · · ·    ← 오른쪽으로 갈수록 폭이 넓어짐(깔때기)
─┼──·················
 │        ·  · · · ·
 │           ·  ·  ·
```

잔차의 **퍼짐이 한쪽에서 더 크면**(예측값이 클수록 오차도 커짐) **이분산**입니다. 계수는 괜찮지만 **표준오차(신뢰도)가 틀어집니다.** → [R5](R5_계수해석_한계효과.md)의 **HC1 강건 표준오차**로 보정합니다. 비유: 작은 가격은 잘 맞히는데 비싼 소일수록 들쭉날쭉하면 깔때기가 생깁니다.

> 즉 **잔차 산점도 한 장**으로 선형성(곡선?)과 등분산성(깔때기?) 두 가정을 동시에 점검합니다.

## 3. Q-Q 플롯 — 정규성 검사

네 번째 가정인 **정규성**(잔차가 종 모양 분포인가)을 보는 전용 그림이 **Q-Q 플롯(Quantile-Quantile plot)** 입니다.

### 아이디어

"내 잔차가 **완벽한 정규분포라면 있어야 할 위치**"와 "**실제 잔차의 위치**"를 짝지어 점으로 찍습니다. 둘이 일치하면 점들이 **대각선 직선 위**에 올라갑니다.

```
실제 잔차
   │              ·
   │           ·          ← 점들이 대각선에 딱 붙으면 정규분포 ✓
   │        ·
   │     ·
   │  ·
   └─────────────────  정규분포라면 있어야 할 위치
```

### 읽는 법

- **점들이 대각선에 붙어 있다** → 정규성 OK.
- **양 끝이 대각선에서 휜다**(보통 S자나 위로 들림) → **꼬리가 두꺼움.** 극단값(아주 비싼/싼 소)이 정규분포보다 많다는 뜻.

> 우리는 이미 [R2](R2_선형회귀_기초.md)에서 가격에 **로그를 씌워** 분포를 폈으므로 정규성이 어느 정도 개선돼 있습니다. 그래도 완벽하진 않을 수 있어 Q-Q로 확인합니다. 다행히 **표본이 매우 크면(122만)** 정규성 위반에 비교적 둔감합니다(중심극한정리 덕분) — 그래도 심하게 휘면 보고서에 적습니다.

## 4. 등분산 점검 — 눈 + 숫자

이분산(깔때기)은 잔차 산점도로 **눈으로** 보지만, **숫자로도** 확인할 수 있습니다. **Breusch-Pagan 검정**이나 **White 검정** 같은 통계 검정이 있습니다.

- 귀무가설: "등분산이다(퍼짐이 일정하다)".
- p값이 작으면(< 0.05) → "등분산 아님 = 이분산" → HC1 강건 표준오차 사용.

> 주의: 표본이 122만이면 이 검정도 **거의 항상 "이분산"이라고 나옵니다**([C1](C1_분류문제와_평가지표.md)·[R5](R5_계수해석_한계효과.md)에서 본 큰 표본의 함정과 같음). 그래서 검정 결과에만 의존하지 말고, **잔차 그림으로 "실제로 얼마나 깔때기인지"** 를 같이 봅니다. 어차피 우리는 안전하게 **처음부터 HC1을 기본으로** 쓰므로([R5](R5_계수해석_한계효과.md)) 이분산이 있어도 대비돼 있습니다.

## 5. 쿡의 거리 — 회귀선을 휘두르는 "위험한 한 마리" 찾기

마지막이자 가장 흥미로운 진단입니다. 가정 검사와는 조금 다른 질문입니다.

> **"단 한 마리의 소가 회귀선 전체를 좌우하고 있지는 않은가?"**

### 영향력 있는 관측치란

대부분의 점은 회귀선에 조금씩 기여합니다. 그런데 **어떤 한 점은 혼자서 회귀선을 확 휘게** 만들 수 있습니다. 보통 ① 다른 소들과 **동떨어진 극단값**이면서 ② **회귀선에서도 멀리** 있을 때 그렇습니다.

```
     ·····/      ← 정상 데이터에 맞춘 회귀선(실선)
   ····/
  ··/
 /                        ●  ← 이 한 점(극단값) 때문에
                              회귀선이 ↗ 이렇게 휘어버림(점선)
```

이런 점은 **지렛대(leverage)** 처럼 회귀선을 들어 올립니다. 한 마리 때문에 "더위 효과"가 통째로 바뀐다면 위험하죠.

### 쿡의 거리(Cook's Distance)

각 관측치에 대해 **"이 점을 빼면 회귀 결과(모든 예측)가 얼마나 바뀌나"** 를 잰 값이 **쿡의 거리**입니다.

- 쿡의 거리가 **크다** = 그 점 하나가 회귀에 **큰 영향** = 주목해야 할 관측치.
- 흔한 경험칙: 4/n 또는 1을 넘으면 "영향력 큼"으로 표시.

### 발견하면 어떻게?

**무조건 지우면 안 됩니다.** 먼저 **왜 그런지** 봅니다([병합·전처리]에서 배운 정신과 같음).
- **입력 오류·이상치**(예: 몸무게 3kg) → 데이터 문제일 수 있음 → 처리 검토.
- **진짜 특수한 소**(예: 노폐우, 초고가 한우) → 실제 데이터 → 함부로 못 지움.

→ 보통 **그 점을 빼고 다시 돌려서 결과가 크게 바뀌는지** 확인합니다(민감도 점검, [R8](R8_강건성과_총정리.md)). 바뀌면 "이 결과는 소수 관측치에 민감하다"고 정직하게 보고합니다.

> 비유: 반 평균 점수를 한 명의 만점(또는 0점)이 좌우한다면, 그 한 명을 빼고도 결론이 같은지 확인해야 "반 전체의 경향"을 믿을 수 있죠. 쿡의 거리는 그 "한 명"을 찾아주는 도구입니다.

## 6. 진단 체크리스트 (한 장 요약)

| 검진 도구 | 보는 가정/문제 | 건강한 모습 | 병들면 처방 |
| --- | --- | --- | --- |
| 잔차 산점도 (곡선?) | 선형성 | 무늬 없음 | 제곱항·비선형 항 추가 |
| 잔차 산점도 (깔때기?) | 등분산성 | 폭 일정 | HC1 강건 표준오차 |
| Q-Q 플롯 | 정규성 | 대각선에 붙음 | 로그 변환(이미 함), 큰 표본이면 관대 |
| 등분산 검정 | 등분산성 | p 큼 | (큰 표본 주의) HC1 |
| 쿡의 거리 | 영향력 이상치 | 다 작음 | 원인 조사 → 빼고 민감도 점검 |

> 모든 진단의 정신은 하나입니다 — **"내 결과를 믿어도 되는지 스스로 의심하고 확인한다."** 이게 우리 프로젝트가 처음부터 강조한 분석 원칙이고, [R8](R8_강건성과_총정리.md)의 강건성 점검으로 이어집니다.

## 7. 핵심 정리

- 회귀 계수는 가정이 깨져도 **숫자로는 나오지만**, 그러면 신뢰도가 틀린다 → **잔차로 건강검진**.
- **잔차 산점도**: 무늬 없으면 건강. **곡선** = 선형성 위반(제곱항), **깔때기** = 이분산(HC1).
- **Q-Q 플롯**: 점이 대각선에 붙으면 정규성 OK. 끝이 휘면 두꺼운 꼬리(큰 표본이면 관대).
- **쿡의 거리**: 회귀선을 혼자 휘두르는 영향력 큰 관측치 탐지. 함부로 지우지 말고 **원인 조사 + 빼고 민감도 점검**.
- 모든 진단의 정신: **"내 결과를 믿어도 되는지 의심하고 확인한다."**
