# C4. 분류 모델 4종 원리 — 로지스틱·LightGBM·XGBoost·CatBoost

> **이 강의의 목표**: 우리가 쓸 네 모델이 각각 **어떤 원리로 예측하는지**를 처음부터 배웁니다. 로지스틱 회귀(선형의 대표), 그리고 결정트리 → 앙상블 → 부스팅으로 이어지는 흐름 위에서 LightGBM·XGBoost·CatBoost가 어떻게 다른지 봅니다. 수식은 최소로, 그림과 비유로.
> **앞 강의**: [00-A](00A_데이터분석이란.md)의 "학습=손잡이 조절", [C3](C3_인코딩과_결측_스케일링.md)의 스케일링·결측(모델마다 다름).


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 분류 4/7**

## 1. 모델을 두 부류로 나눠 보기

분류 모델은 수백 가지지만, 우리 4개는 **두 부류**에 속합니다. 이 구분을 먼저 잡으면 나머지가 쉽습니다.

- **선형 모델**: 변수들을 **직선(평면)으로** 가른다. → **로지스틱 회귀**
- **트리 기반 부스팅**: **계단처럼 쪼개가며** 복잡한 경계를 그린다. → **LightGBM, XGBoost, CatBoost**

비유: 밭에서 잡초와 작물을 가른다면 —
- 선형: "이 **대각선** 왼쪽은 잡초"라고 **직선 하나**로 가름. 단순·빠름·해석 쉬움. 하지만 경계가 구불구불하면 못 따라감.
- 트리: "여기서 자르고, 또 저기서 자르고…" **계단 모양**으로 복잡하게 가름. 구불구불한 경계도 잡음. 대신 과하면 외움(과적합).

## 2. 로지스틱 회귀 — 선형 모델의 대표 (베이스라인)

이름에 "회귀"가 있지만 **분류 모델**입니다(헷갈리는 이름). 우리는 이걸 **베이스라인(기준점)** 으로 씁니다 — "가장 단순한 모델로 이 정도는 나온다"의 바닥선.

### 어떻게 분류하나

1단계, 변수들을 가중치로 **직선 결합**해 점수를 냅니다.

```
점수 = w₁·몸무게 + w₂·월령 + w₃·더위 + ... + b
```

2단계, 이 점수(−∞~+∞)를 **0~1 확률**로 변환합니다. 여기에 **시그모이드(sigmoid)** 함수를 씁니다.

```
확률 = 1 / (1 + e^(−점수))      ← S자 곡선
```

> **`e`가 뭔가요?** `e`는 **자연상수**라 부르는 고정된 숫자(약 2.718)입니다. 원주율 π가 3.14…로 정해진 상수이듯, e도 그냥 정해진 값입니다. 수학에서 "매끄럽게 늘어나고 줄어드는" 곡선을 만들 때 자연스럽게 등장해서 시그모이드·소프트맥스·로그 같은 데 늘 쓰입니다. 지금은 **"e^(어떤 수)는 그 수를 매끄러운 곡선으로 바꿔주는 버튼"** 정도로만 알면 충분합니다.

점수가 크면 확률이 1에 가깝고, 작으면 0에 가깝습니다. 0.5를 넘으면 "그 클래스"라고 판정. 이게 로지스틱 회귀입니다.

### 여러 클래스는? — 소프트맥스

우리는 16개 등급이라 2개짜리 시그모이드로는 부족합니다. 클래스가 여러 개일 땐 **소프트맥스(softmax)** 를 씁니다.

```
P(등급=k) = e^(점수ₖ) / (모든 등급의 e^(점수) 합)
```

각 등급의 점수를 e로 올린 뒤 비율로 나눠, **16개 확률을 합이 1이 되게** 만듭니다. 가장 확률 높은 등급을 답으로 냅니다. (sklearn 1.x는 이걸 자동으로 합니다 — 옛날엔 `multi_class` 인자로 지정했지만 이제 불필요.)

### 특징

- **장점**: 빠르고, 계수(w)로 "어느 변수가 어느 방향으로 작용하는지" **해석**이 쉽다.
- **단점**: 직선 경계만 그려서 복잡한 패턴은 약하다. **스케일링 필수**([C3](C3_인코딩과_결측_스케일링.md)), 결측 못 먹음(채워야 함).
- **우리 용도**: 베이스라인. 트리 모델이 "이 단순한 것보다 얼마나 나은가"를 재는 자.

## 3. 결정트리 — 부스팅을 이해하기 위한 기초

LightGBM/XGBoost/CatBoost를 이해하려면 먼저 **결정트리(Decision Tree)** 하나를 알아야 합니다.

### 스무고개 게임

결정트리는 **스무고개**와 똑같습니다. 질문을 던져가며 답을 좁힙니다.

```
                [몸무게 > 450?]
                /            \
              예              아니오
              │                │
        [월령 > 30?]      [성별=거세?]
         /      \           /      \
      1++급    1+급        2급      3급
```

각 갈림에서 **"어느 질문이 데이터를 가장 깔끔하게 둘로 가르나"** 를 따져 질문을 고릅니다. 잎(맨 끝)에 도달하면 거기 모인 소들의 다수 등급으로 예측합니다. 그런데 "가장 깔끔하게"를 무엇으로 재는지가 트리의 핵심이라, 따로 봅니다.

### 트리는 "불순도"로 질문을 고른다

"깔끔하다"를 수치로 잰 것이 **불순도(impurity)** 입니다. 직관은 단순합니다.

- 한 그룹에 **한 등급만** 모여 있으면 → **순수함 = 불순도 0** (완벽히 갈림)
- 여러 등급이 **골고루 섞여** 있으면 → **불순도 높음** (뒤죽박죽)

트리는 매 갈림에서 **"이 질문으로 쪼개면 불순도가 얼마나 줄어드나"** 를 모든 후보 질문에 대해 계산하고, **불순도를 가장 많이 줄이는 질문**을 고릅니다. 비유하자면, 빨강·파랑 구슬이 섞인 상자를 칸막이 하나로 나눌 때 **"한쪽은 빨강, 다른 쪽은 파랑"에 가장 가깝게** 갈라주는 칸막이를 고르는 것입니다.

**간단한 예시.** 소 10마리(1등급 5, 2등급 5 — 완전 뒤죽박죽)를 두 질문으로 쪼개본다고 합시다.

```
질문 A "몸무게>450?":   예[1등급4, 2등급1]  /  아니오[1등급1, 2등급4]
   → 양쪽 다 한 등급으로 거의 쏠림 = 불순도 많이 줄어듦 → 좋은 질문 ✓

질문 B "태어난 달이 짝수?":  예[1등급3, 2등급2]  /  아니오[1등급2, 2등급3]
   → 양쪽 다 여전히 반반 = 불순도 거의 그대로 → 쓸모없는 질문 ✗
```

트리는 질문 A를 고릅니다. 불순도를 더 많이 줄였으니까요.

불순도를 재는 구체적 공식은 두 가지가 흔합니다(이름만 알면 됩니다).
- **지니 불순도(Gini)**: "이 그룹에서 무작위로 둘을 뽑았을 때 등급이 서로 다를 확률". 0이면 다 같은 등급(순수). 계산이 빨라 기본값으로 많이 씀.
- **엔트로피(entropy)**: 정보이론에서 온 "무질서한 정도". 0이면 순수, 섞일수록 큼. 지니와 결과는 거의 비슷.

> 한 줄 요약: **트리는 "불순도를 가장 많이 줄이는 질문"을 매 단계 골라가며 데이터를 점점 순수한 그룹으로 쪼갠다.** 지니·엔트로피는 그 불순도를 재는 두 가지 자(尺)일 뿐입니다.

### 장점과 치명적 단점

- **장점**: 직관적(사람이 읽을 수 있음), 스케일링 불필요, 결측도 처리 가능, 비선형 패턴 OK.
- **치명적 단점**: 트리 하나는 **과적합을 너무 잘합니다.** 깊게 키우면 학습 데이터를 통째로 외워버려서 새 데이터에 약합니다.

이 단점을 푸는 게 **앙상블**입니다.

## 4. 앙상블 — 여럿이 모이면 똑똑해진다

**앙상블(ensemble)** = 여러 모델의 예측을 합쳐 하나의 더 좋은 예측을 만드는 것. "집단지성"입니다. 트리 앙상블엔 두 방식이 있습니다.

### 방식 ① 배깅(Bagging) — 병렬로 여럿, 투표

데이터를 조금씩 다르게 뽑아 **트리 여러 개를 따로따로** 키우고, **투표(또는 평균)** 로 결정. 대표가 **랜덤 포레스트(Random Forest)** 입니다. 트리 하나는 과적합해도 여럿의 평균은 안정적이죠. "전문가 100명에게 따로 묻고 다수결".

### 방식 ② 부스팅(Boosting) — 직렬로, 실수를 이어 고침 (우리 3종!)

이게 우리 모델들의 방식이라 중요합니다. **트리를 하나씩 순서대로** 만들되, **앞 트리가 틀린 부분을 다음 트리가 집중적으로 고칩니다.**

```
트리1: 일단 예측 → 어디서 많이 틀렸는지 본다
트리2: 트리1이 틀린 곳을 집중 보정
트리3: 트리1+2가 아직 틀린 곳을 또 보정
...
수백 개를 쌓으면 → 점점 정확해짐
```

비유: 시험을 보고 **오답노트를 만들어 약점만 집중 보완**하는 걸 수백 번 반복. "틀린 것에 집중"이 핵심이라, 배깅보다 보통 더 정확합니다(대신 과적합·튜닝에 더 신경 써야 함).

### 그래디언트 부스팅 — "얼마나 틀렸나"의 방향으로

우리 3종은 정확히는 **그래디언트 부스팅(Gradient Boosting)** 입니다. "틀린 곳"을 수학적으로 **손실의 기울기(gradient)** 로 잡아, 그 방향으로 다음 트리를 맞춥니다.

```
다음 트리 = 현재까지의 예측이 "어느 방향으로 얼마나" 틀렸는지(잔차/기울기)를 맞히는 트리
최종 예측 = 트리1 + (학습률 × 트리2) + (학습률 × 트리3) + ...
```

여기서 **학습률(learning_rate)** 은 "한 걸음의 보폭"입니다. 작으면 천천히 정밀하게(트리가 더 많이 필요), 크면 빠르지만 거칠게. (C5 튜닝의 핵심 손잡이.)

## 5. 부스팅 3형제 — 같은 원리, 다른 성격

LightGBM·XGBoost·CatBoost는 **다 그래디언트 부스팅**입니다. 원리는 같고 **"트리를 어떻게 키우느냐"의 전략**이 다릅니다. 형제처럼 닮았지만 성격이 달라요.

### XGBoost — 정석, 꼼꼼한 맏형

부스팅을 대중화한 모델. **레벨 단위(level-wise)** 로 트리를 키웁니다 — 같은 깊이의 가지를 **고르게** 펼칩니다. 과적합 억제 장치(정규화)가 목적함수에 잘 녹아 있어 **안정적**입니다. 균형 잡힌 정석.

### LightGBM — 빠른 둘째, 우리 메인

**잎 단위(leaf-wise)** 로 키웁니다 — 고르게가 아니라 **"가장 많이 틀린 잎"부터** 깊게 파고듭니다. 그래서 **빠르고**(대용량에 강함) 성능도 좋습니다. 우리 240만 행에 적합해서 **메인 모델**로 씁니다. 단, 깊게 파다 과적합할 수 있어 잎 개수(num_leaves) 등을 잘 잡아야 합니다.

### CatBoost — 범주형의 막내

이름이 **Cat**egorical Boosting — **범주형 변수를 자동으로 잘 처리**하도록 설계됐습니다. **대칭 트리(symmetric)** 를 써서 모든 가지를 같은 기준으로 쪼개 **과적합에 강하고** 안정적입니다. "순서형 부스팅"이라는 기법으로 타깃 인코딩류의 누수도 잘 막습니다. 대신 상대적으로 느린 편.

### 비교 표 (핵심)

| | XGBoost | LightGBM | CatBoost |
| --- | --- | --- | --- |
| 트리 키우는 법 | 레벨 단위(고르게) | **잎 단위(깊게)** | 대칭 트리 |
| 속도 | 빠름 | **가장 빠름** | 느린 편 |
| 강점 | 안정·정규화 | 대용량·기본성능 | 범주형·과적합 강함 |
| 결측 처리 | 자동 | 자동 | 자동 |
| 우리 역할 | 교차 확인 | **메인** | 교차 확인 |

> 세 형제를 다 돌려보는 이유: **모델마다 잘 틀리는 부분이 달라서**, 비교하면 어느 게 우리 데이터에 맞는지 알 수 있고, 나중에 **셋의 예측을 합치면(앙상블)** 더 좋아지기도 합니다(C7).

## 6. 왜 트리 부스팅이 표 데이터의 1순위인가

캐글(데이터 대회)에서 **표(엑셀) 형태 데이터**는 거의 항상 부스팅이 우승합니다. 이유:

1. **비선형·상호작용을 자동으로** 잡습니다(트리가 계단으로 그리니까).
2. **스케일링·결측 처리가 거의 필요 없습니다**([C3](C3_인코딩과_결측_스케일링.md)).
3. 변수 단위가 제각각이어도 잘 작동합니다.
4. 적당히 튜닝하면 안정적으로 높은 성능.

(딥러닝은 이미지·텍스트엔 강하지만, 표 데이터에선 보통 부스팅을 못 이깁니다. 그래서 우리도 부스팅이 메인입니다.)

## 7. 핵심 정리

- 우리 4모델 = **선형 1개(로지스틱)** + **트리 부스팅 3개(LGB/XGB/Cat)**.
- **로지스틱**: 직선 결합 → 시그모이드/소프트맥스로 확률. 빠르고 해석 쉬움, 스케일링 필수. **베이스라인**.
- **결정트리**: 스무고개로 쪼갬. 직관적이지만 혼자선 과적합.
- **앙상블**: 배깅(병렬·투표=랜덤포레스트) vs **부스팅(직렬·오답 보정=우리 3종)**.
- **그래디언트 부스팅**: 앞 트리의 틀린 방향(기울기)을 다음 트리가 보정. 학습률 = 보폭.
- **3형제**: XGB(정석·안정), **LGB(빠름·메인)**, Cat(범주형·과적합 강함). 원리 같고 전략 다름.
- 표 데이터의 1순위가 부스팅인 이유: 비선형·결측·단위에 강하고 성능 안정.
