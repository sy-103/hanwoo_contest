# R5. 계수 해석과 한계효과 — 숫자를 보고서 문장으로

> **이 강의의 목표**: 회귀가 뱉은 계수를 **"더위 1%p↑ → 가격 N% 변화"** 같은 보고서 문장으로 옮기는 법을 배웁니다. 로그 계수의 정확한 퍼센트 변환, 변수끼리 영향력 비교(표준화), 더미 계수 해석, 그리고 그 효과를 얼마나 믿을지 재는 강건 표준오차까지. **회귀팀 보고서의 핵심 산출물**이 여기서 나옵니다.
> **앞 강의**: [R2](R2_선형회귀_기초.md)의 계수·로그 가격, [R1](R1_회귀의_목적_인과.md)의 통제·총효과, [R4](R4_규제회귀_라쏘릿지.md)(해석은 OLS로).


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 회귀 5/8**

## 1. 계수 = 회귀의 최종 답

[R1](R1_회귀의_목적_인과.md)에서 "회귀의 산출물은 점수가 아니라 문장"이라 했죠. 그 문장의 재료가 **계수**입니다. 계수 하나를 제대로 읽고 쓰는 게 회귀팀 일의 90%입니다. 단계적으로 배웁니다.

기본 회귀(로그 가격):

```
log(가격) = b₀ + b₁·더위 + b₂·월령 + ... + (지역·계절 더미)
```

각 `b`를 어떻게 문장으로 옮기느냐가 이 강의 전체입니다.

---

## 2. 로그 계수를 퍼센트로 — 정확한 공식

[R2](R2_선형회귀_기초.md)에서 "로그 가격을 쓰면 계수가 퍼센트 해석이 된다"고 했죠. 정확히 합시다.

```
log(가격) 모델에서 변수가 1 늘 때 가격 변화율:

  정확한 공식:  (e^b − 1) × 100 %      ← 보고서엔 이걸 쓴다
  간단한 근사:  b × 100 %               ← |b|가 작을 때만(0.1 이하) 거의 같음
```

### 왜 근사가 아니라 정확식을 써야 하나

작은 계수면 둘이 거의 같지만, **계수가 크면 크게 틀립니다.**

| 계수 b | 근사(b×100) | 정확((e^b−1)×100) | 차이 |
| --- | --- | --- | --- |
| −0.008 | −0.8% | −0.797% | 거의 같음 ✓ |
| 0.10 | +10% | +10.5% | 작은 차이 |
| **0.40** | **+40%** | **+49.2%** | **9%p나 틀림!** |

**특히 더미 변수**(거세 vs 암 같은)는 계수가 크기 쉬워서, 근사를 쓰면 크게 틀립니다. 그래서 **항상 정확식 (e^b−1)×100%** 를 씁니다.

### 실제 문장 만들기

더위 변수(ratio_고온)는 0~1 비율이라 "1 증가"는 비현실적입니다(0%→100%). 그래서 **현실적인 단위**로 환산합니다.

```
계수 b = 0.29 (예시)
더위 노출이 1%p(=0.01) 증가하면:
  (e^(0.29 × 0.01) − 1) × 100 = +0.29%

→ 문장: "생애 더위 노출이 1%p 늘면, 다른 조건이 같을 때 가격이 약 0.29% 오른다"
```

> **"다른 조건이 같을 때"를 절대 빼지 마세요.** [R2](R2_선형회귀_기초.md)에서 봤듯 다중회귀 계수는 "다른 변수를 고정한" 효과입니다. 이 말을 빼면 인과처럼 들리는 과장이 됩니다.

## 3. 변수끼리 영향력 비교 — 표준화 계수(1-SD 효과)

또 다른 질문: **"더위와 월령 중 누가 가격에 더 큰 영향을 주나?"** 계수를 그냥 비교하면 안 됩니다. **단위가 달라서**입니다.

```
더위 비율: 0 ~ 1 범위        → 계수가 크게 나옴
월령:      28 ~ 36 범위      → 계수가 작게 나옴
   → 계수 크기로 "누가 중요"를 비교하면 단위에 속는다
```

해결: 각 변수를 **"1 표준편차(1-SD) 움직였을 때"** 의 효과로 환산해 비교합니다. "그 변수가 현실에서 흔들리는 폭만큼 움직이면 가격이 몇 % 변하나"라는 공정한 잣대죠.

```
1-SD 효과 = (e^(b × 그 변수의 표준편차) − 1) × 100 %
```

이 값의 절댓값이 클수록 영향력 큰 변수입니다. 이걸로 "영향력 순위" 막대그래프를 그립니다.

> 정리: **두 가지 계수를 둘 다 보고합니다.**
> - **1단위 효과** (e^b−1)×100% → "더위 1%p당 N%" 같은 **구체적 해석**.
> - **1-SD 효과** → 변수끼리 **영향력 크기 비교**.

---

## 4. 더미 계수는 "기준 대비"로 읽는다

[C3](C3_인코딩과_결측_스케일링.md)에서 원-핫 할 때 `drop_first`로 한 범주를 기준으로 뺐죠. 그래서 **더미 계수는 항상 "그 기준 범주 대비"** 입니다.

```
sido_제주 계수 = −0.1 이고, 기준이 강원도라면:
  → "제주는 강원도 대비 가격이 (e^(−0.1)−1)×100 ≈ −9.5%"
  (제주의 절대 가격이 아니라, 강원 대비 차이!)
```

**기준 범주를 안 밝히면 틀린 문장**이 됩니다. 우리 코드에선 시도는 강원, 계절은 가을, 성별은 거세가 기준입니다(drop_first가 빼는 첫 범주). 보고할 때 "강원 대비", "거세 대비"를 꼭 붙입니다.

## 5. 효과를 얼마나 믿나 — 표준오차와 신뢰구간

계수 값만으로는 부족합니다. **"이 효과를 믿어도 되나?"** 를 같이 봐야 합니다. 여기서 세 가지가 나옵니다.

### 표준오차(SE) — 계수의 흔들림

같은 분석을 데이터를 조금씩 바꿔가며 반복하면 계수가 조금씩 달라집니다. 그 **흔들리는 폭**이 표준오차입니다. 작을수록 "이 계수를 믿을 만하다".

### 신뢰구간(CI) — 효과의 범위

"더위 효과가 정확히 −0.8%"가 아니라 **"95% 확신하건대 −1.2% ~ −0.4% 사이"** 라고 범위로 말하는 게 정직합니다. 이 범위가 **95% 신뢰구간**입니다.

```
보고 문장: "더위 1%p↑ → 가격 −0.8% (95% CI: −1.2% ~ −0.4%)"
```

> **중요**: 신뢰구간이 **0을 포함하면** 방향조차 단정 못 합니다(+일 수도 −일 수도). 0을 포함 안 하면 "확실히 음(또는 양)의 효과"라고 말할 수 있습니다.

### p값 — 그런데 우리 데이터에선 주의

p값은 "이 효과가 우연일 확률"입니다. 보통 0.05 미만이면 "유의하다"고 하죠. **그런데 우리는 표본이 122만 개**라, 아주 작은 효과도 거의 다 p<0.05로 나옵니다. 그래서 **p값(별 개수)보다 효과의 크기(%)와 신뢰구간 폭**으로 이야기해야 합니다. "p<0.001이니 중요하다"는 이 표본 크기에선 공허한 말입니다.

## 6. 표준오차를 "강건하게" — 가정이 깨질 때의 보강

[R2](R2_선형회귀_기초.md)에서 회귀의 4가정(선형·독립·등분산·정규)을 배웠죠. 이게 깨지면 **계수는 괜찮은데 표준오차(신뢰도)가 틀립니다.** 우리 데이터는 두 가정이 의심스러워서 보강합니다.

### 이분산 → 강건 표준오차(HC1)

가격의 흔들림 폭이 가격대마다 다를 수 있습니다(등분산 위반 = 이분산). 그러면 일반 표준오차가 틀립니다. **HC1 강건 표준오차**는 이분산이 있어도 올바른 신뢰도를 줍니다. 계수는 그대로, 표준오차만 보정.

```python
sm.OLS(y, X).fit(cov_type="HC1")   # 계수 동일, 표준오차만 강건하게
```

### 농장 내 상관 → 클러스터 표준오차

같은 농장 소들은 오차가 서로 닮았을 수 있습니다(독립성 위반). **농장 클러스터 표준오차**는 이걸 반영해 더 보수적인(넓은) 신뢰구간을 줍니다.

> **강건성 점검**: HC1과 클러스터 표준오차에서 **결론(유의성)이 같으면** 결과가 견고하다는 추가 증거입니다. 클러스터에서만 유의성이 사라지면 "농장 수준 현상일 수 있다"고 보고서에 솔직히 적습니다.

## 7. 총효과 vs 직접효과 — 매개 분해를 보고서로

[R1](R1_회귀의_목적_인과.md)에서 배운 매개변수(몸무게·증체)를 여기서 활용합니다.

```
총효과 모델   (몸무게·증체 제외): 더위 1%p → 가격 +0.29%  ← 헤드라인
직접효과 모델 (몸무게·증체 포함): 더위 1%p → 가격 +0.23%
                                        ─────────
간접효과 (더위가 체격을 통해)   : 약 +0.06%p
```

보고서엔 **총효과를 메인 숫자**로, 그 차이를 **"더위가 체중·증체를 통해 주는 간접 경로"** 로 부연합니다. 이렇게 하면 "더위가 가격에 영향을 주는 *방식*"까지 설명하는 풍부한 결과가 됩니다.

## 8. 보고서 문장 만들기 — 완성형 템플릿

지금까지 배운 걸 합쳐 한 문장을 만듭니다.

```
[변수]가 [현실적 단위]만큼 변하면,
같은 [통제한 조건들]의 소 대비
가격이 약 [N]% ([95% CI: a~b]) [증가/감소]한다.
```

예: *"생애 더위 노출이 1%p 늘면, 같은 지역·계절·연도·성별·월령의 소 대비 가격이 약 0.29% (95% CI: 0.21~0.37%) 오른다."*

여기에 **마리당 금액 환산**을 더하면 실감 납니다: *"평균 도체중 423kg, 평균가 16,662원/kg 기준, 더위 5%p 증가 시 마리당 약 OO원."* (단, 부호가 양수로 나오면 [R1](R1_회귀의_목적_인과.md)의 교란을 다시 의심 — "더위가 좋다"가 아니라 남은 교란일 수 있음.)

## 9. 핵심 정리

- 회귀의 최종 답은 **계수 → 문장**. "다른 조건이 같을 때 [변수] → 가격 N%".
- 로그 계수의 퍼센트 변환은 **정확식 (e^b−1)×100%** (근사 b×100은 큰 계수에서 틀림, 특히 더미).
- 두 계수를 보고: **1단위 효과**(구체적 해석) + **1-SD 효과**(영향력 비교).
- **더미 계수는 "기준 범주 대비"** — 기준(강원/가을/거세)을 꼭 밝힌다.
- 믿음은 **표준오차·신뢰구간**으로. CI가 0을 포함하면 방향 단정 못 함. **p값보다 크기·CI**(표본 122만).
- 가정 깨짐 대비: **HC1**(이분산), **클러스터**(농장 내 상관). 둘 결론 같으면 견고.
- **총효과(메인) vs 직접효과** 비교로 간접 경로까지 설명.
