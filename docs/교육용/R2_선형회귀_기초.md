# R2. 선형회귀의 원리 — 직선을 어떻게 긋나

> **이 강의의 목표**: 선형회귀(OLS)가 직선을 **어떻게** 긋는지(최소제곱), 계수가 **무슨 뜻**인지, 가격에 **로그를 왜** 씌우는지, 그리고 회귀가 **무엇을 가정**하는지를 처음부터 배웁니다. 공식이 나오지만 전부 직관으로 풀어 설명합니다.
> **앞 강의**: [R1](R1_회귀의_목적_인과.md)의 "회귀=관계식, 계수 해석이 핵심".


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 회귀 2/8**

## 1. 직선 하나로 시작하기

변수가 하나일 때(더위 → 가격)를 먼저 봅시다. 점들이 흩어진 그래프에 **직선 하나**를 긋는 게 회귀입니다.

```
가격
 │        ·   ·
 │     ·  ·· ·   ← 점: 소 한 마리 (더위, 가격)
 │   ·· · ·  ·
 │  ──────────── ← 회귀 직선: 점들을 가장 잘 대표하는 선
 │ ·· ·
 └──────────────── 더위
```

직선의 식: `가격 = b₀ + b₁ × 더위`

- **b₀ (절편)**: 더위가 0일 때의 가격(직선이 세로축과 만나는 높이).
- **b₁ (기울기/계수)**: **더위가 1 늘면 가격이 b₁만큼 변한다.** ← **이게 우리가 알고 싶은 그 숫자.**

[R1](R1_회귀의_목적_인과.md)에서 "계수 해석이 회귀의 전부"라 한 게 이 b₁입니다. b₁이 음수면 "더위가 늘수록 가격이 떨어진다", 양수면 "오른다"는 뜻이죠.

---

## 2. "가장 잘 대표하는" 직선은 어떻게 찾나 — 최소제곱법

직선은 무한히 그을 수 있는데, 그중 **"가장 좋은"** 직선을 어떻게 고를까요? 기준이 필요합니다.

### 잔차(residual) — 빗나간 정도

각 점에서 직선까지의 **세로 거리**가 "그 소를 얼마나 못 맞췄나"입니다. 이걸 **잔차**라 합니다.

```
잔차 = 실제 가격 − 직선이 예측한 가격
```

잔차가 작을수록 직선이 그 점을 잘 맞춘 거죠. 모든 점의 잔차가 **전체적으로 작은** 직선이 좋은 직선입니다.

### 왜 "제곱"인가

잔차를 그냥 더하면 양수(+)와 음수(−)가 상쇄돼 0이 됩니다(위로 빗나간 점과 아래로 빗나간 점이 비김). 그래서 **제곱해서 더합니다.** 제곱하면 다 양수가 되고, 큰 빗나감에 더 큰 벌점이 갑니다.

```
최소제곱법(OLS, Ordinary Least Squares):
  잔차 제곱의 합 Σ(실제 − 예측)² 을 가장 작게 만드는 b₀, b₁을 찾는다
```

이게 **OLS**입니다. 이름 그대로 "잔차 제곱(Least Squares)을 최소화". 컴퓨터가 미적분으로 단번에 정답을 계산합니다(분류의 부스팅처럼 반복하지 않고 공식으로). 변수가 여러 개여도 원리는 같습니다 — 직선이 평면·초평면이 될 뿐.

> 외울 점: **OLS = "빗나간 정도(잔차)의 제곱 합을 최소로 하는 선."** 그게 다입니다.

---

## 3. 변수가 여러 개면 — 다중 회귀

실제론 더위만 보지 않습니다. [R1](R1_회귀의_목적_인과.md)에서 교란변수(지역·계절…)를 같이 넣어야 한다고 했죠. 그래서:

```
가격 = b₀ + b₁·더위 + b₂·몸무게 + b₃·월령 + b₄·지역더미 + ...
```

이때 각 계수의 해석이 미묘하게 바뀝니다. **b₁ = "다른 변수들을 다 고정했을 때, 더위가 1 늘면 가격이 변하는 양."** 즉 **"같은 지역·같은 월령 조건에서의 더위 효과"** — 이게 바로 [R1](R1_회귀의_목적_인과.md)의 "통제"가 수식으로 일어나는 방식입니다. 다른 변수를 넣는 것만으로 자동으로 "그 변수를 고정한 효과"가 됩니다.

> 그래서 회귀 결과를 말할 땐 항상 **"다른 조건이 같을 때"** 를 붙입니다. 이걸 빼면 인과처럼 들리는 과장이 됩니다.

---

## 4. 가격에 로그를 씌우는 이유

우리 회귀팀은 가격 그대로가 아니라 **로그를 씌운 값**(`log_cost = ln(가격)`)을 타깃으로 씁니다. 왜일까요? 두 가지 이유가 있습니다.

### 이유 ① 분포를 펴기

가격 분포는 **오른쪽으로 길게 치우쳐** 있습니다(대부분 1~2만원대인데 일부 비싼 소가 꼬리를 길게 끌죠). 이런 치우친 분포는 회귀 가정(아래 6절)에 안 맞습니다. **로그를 씌우면 이 꼬리가 눌려서 종 모양에 가까워집니다.**

```
원래 가격:    |▆▅▃▂▁_____  (오른쪽 꼬리 길다)
로그 가격:    __▂▅▆▅▂__    (종 모양에 가까워짐)
```

### 이유 ② 해석이 "퍼센트"로 바뀐다 (이게 더 중요)

로그를 씌우면 계수의 의미가 **"몇 원"이 아니라 "몇 퍼센트"** 로 바뀝니다. 이게 훨씬 유용합니다 — "더위가 가격을 800원 낮춘다"보다 "더위가 가격을 0.8% 낮춘다"가 비교·해석에 좋으니까요.

```
log(가격) = b₀ + b₁·더위  일 때,
  더위가 1 늘면 가격이 약 (b₁ × 100)% 변한다
  (정확히는 (e^b₁ − 1)×100% — R5에서 자세히)
```

> 직관: 로그 세계에서의 "덧셈"은 원래 세계에서 "곱셈(비율)"입니다. 그래서 로그 타깃의 계수는 자연히 "비율 변화 = 퍼센트"가 됩니다. R5에서 이 변환을 정확한 공식으로 다룹니다.

## 5. R² — 모델이 얼마나 설명하나

회귀가 "얼마나 잘 맞나"를 재는 대표 지표가 **R²(결정계수)** 입니다.

```
R² = 모델이 설명한 변동 / 전체 변동      (0 ~ 1)
```

- R² = 1: 점들이 직선 위에 완벽히 있음(다 설명).
- R² = 0: 직선이 아무것도 설명 못 함(그냥 평균 긋는 거랑 같음).
- R² = 0.4: 가격 변동의 40%를 이 변수들로 설명.

> **주의 (회귀팀 필독)**: 우리 목적은 [R1](R1_회귀의_목적_인과.md)에서 말했듯 **예측이 아니라 효과 추정**입니다. 그래서 **R²가 낮아도 괜찮습니다.** 개체 가격엔 경매 당일 수급 같은 모델 밖 요인이 많아 R²가 0.4 수준이어도, 계수(더위 효과)가 안정적이고 통계적으로 유의하면 충분히 의미 있는 분석입니다. 반대로 **R²가 비정상적으로 높으면 오히려 누수**(매개·정답 변수가 새어 들어감)를 의심해야 합니다.

## 5.5 회귀 평가지표 모음 — RMSE · MAE · Adjusted R²

R²만으로는 부족합니다. 실습 코드(20회차 셀 12)는 **RMSE**도 출력하니, 회귀의 평가지표 네 개를 한 번에 정리합니다. 분류의 [C1](C1_분류문제와_평가지표.md)이 정확도·F1을 배웠듯, 회귀에도 "잘했나"를 재는 여러 자가 있습니다.

먼저 공통 재료, **잔차(오차)** = 실제값 − 예측값. 모든 지표가 이 오차를 어떻게 요약하느냐의 차이입니다.

### MAE (평균 절대 오차) — "평균적으로 얼마나 빗나가나"

```
MAE = 평균( |실제 − 예측| )      ← 오차의 절댓값 평균
```

가장 직관적입니다. "예측이 평균 OOO원(또는 로그값 OO) 빗나간다"로 바로 읽힙니다. 절댓값이라 **모든 오차를 똑같이** 취급합니다 — 1만원 빗나간 것 10개와 10만원 빗나간 것 1개를 같은 무게로 봅니다.

### RMSE (평균 제곱근 오차) — "큰 실수에 민감한 자"

```
RMSE = √( 평균( (실제 − 예측)² ) )      ← 오차를 제곱→평균→제곱근
```

오차를 **제곱**하는 게 핵심입니다([R2](R2_선형회귀_기초.md) OLS와 같은 정신). 제곱하면 **큰 오차에 훨씬 큰 벌점**이 갑니다. 그래서 RMSE는 **가끔 크게 틀리는 것을 특히 싫어합니다.**

### MAE vs RMSE — 언제 무엇을 보나 (핵심 직관)

비유: 배달 시간 예측. 보통 ±5분인데 **딱 한 번 60분 늦었다고** 합시다.
- **MAE**: 그 60분도 다른 오차와 똑같이 1표 → 평균이 별로 안 흔들림("대체로 정확").
- **RMSE**: 60분을 제곱해 크게 벌점 → 값이 확 뜀("가끔 크게 틀린다"를 경고).

```
RMSE는 항상 MAE보다 크거나 같다.
둘의 격차가 크면 → "가끔 아주 크게 빗나가는 예측(큰 이상치 오차)이 있다"는 신호.
둘이 비슷하면 → 오차가 고르게 퍼져 있다.
```

→ **큰 실수가 치명적인 상황이면 RMSE**, **전반적 정확도가 궁금하면 MAE**를 봅니다. 둘을 같이 보면 "평균적으로 얼마나, 그리고 가끔 얼마나 크게" 틀리는지 둘 다 알 수 있습니다.

### R² vs Adjusted R² — 변수를 늘리면 R²는 반칙으로 오른다

R²엔 함정이 하나 있습니다. **쓸모없는 변수를 아무거나 추가해도 R²는 절대 안 내려가고, 보통 조금이라도 오릅니다.** 우연한 잡음까지 조금씩 맞히기 때문이죠. 그래서 변수를 잔뜩 넣으면 R²가 "공짜로" 높아 보이는 착시가 생깁니다.

**Adjusted R²(수정된 결정계수)** 는 이 반칙을 막습니다 — **변수를 늘릴 때마다 벌점**을 매겨서, 그 변수가 **벌점 이상으로 기여할 때만** 값이 오르게 합니다.

```
Adjusted R² = 1 − (1−R²) × (n−1)/(n−k−1)
                                  ↑ k(변수 수)가 늘면 분모가 작아져 값이 깎임
   (n = 표본 수, k = 변수 수)
```

> 직관: R²는 "변수를 넣을수록 무조건 후해지는 채점", Adjusted R²는 "쓸모없는 변수를 넣으면 오히려 깎는 엄격한 채점". **두 모델의 변수 개수가 다를 때 공정하게 비교**하려면 Adjusted R²를 봅니다. (변수 수가 같으면 둘은 거의 같습니다.)

### 정리 표

| 지표 | 무엇을 재나 | 특징 | 좋은 값 |
| --- | --- | --- | --- |
| **MAE** | 평균 절대 오차 | 모든 오차 동등, 직관적 | 작을수록 |
| **RMSE** | 제곱 평균 오차 | 큰 오차에 민감 | 작을수록 |
| **R²** | 설명한 변동 비율 | 변수 늘면 반칙으로 오름 | 1에 가까울수록 |
| **Adjusted R²** | 변수 수 벌점 준 R² | 모델 간 공정 비교용 | 1에 가까울수록 |

> 우리 트랙2는 **효과 추정**이 목적이라 이 지표들(예측 정확도)은 **보조**입니다([R1](R1_회귀의_목적_인과.md)). 그래도 모델끼리 비교하거나(R4 규제 모델 vs OLS) 과적합을 점검할 때 RMSE·Adjusted R²를 봅니다. 메인은 어디까지나 **계수와 신뢰구간**([R5](R5_계수해석_한계효과.md))입니다.

## 6. 회귀의 4가지 가정 (믿어도 되는지의 조건)

OLS 계수를 **믿으려면** 몇 가지 전제가 맞아야 합니다. 처음엔 이름만 알아두고, 21회차 잔차 진단에서 실제로 점검합니다.

| 가정 | 뜻 | 어기면 |
| --- | --- | --- |
| **선형성** | 관계가 대체로 직선이다 | 곡선인데 직선 그으면 효과를 잘못 잼 |
| **독립성** | 한 소의 오차가 다른 소와 무관 | 같은 농장 소끼리 닮으면 위반 → 표준오차 틀림 |
| **등분산성** | 오차의 퍼짐이 어디서나 일정 | 가격대마다 퍼짐 다르면(이분산) p값 틀림 |
| **정규성** | 오차가 종 모양 분포 | 심하게 치우치면 추론 부정확 (로그로 완화) |

> 이 가정들이 깨지면 **계수 자체보다 "표준오차(신뢰도)"가 틀어집니다.** 우리는 이에 대비해 R5에서 **강건 표준오차(HC1)·농장 클러스터** 같은 보강책을 씁니다. 지금은 "회귀엔 전제가 있고, 그게 깨지면 신뢰도가 흔들린다"만 기억하세요.

## 7. 분류와 회귀, 학습 방식의 차이

[C4](C4_분류모델_4종.md)의 부스팅과 비교하면 회귀의 성격이 분명해집니다.

| | 부스팅(분류) | 선형회귀(OLS) |
| --- | --- | --- |
| 학습 방식 | 트리를 반복해 쌓음 | **공식으로 한 번에** 계산 |
| 경계 | 복잡한 계단 | 직선/평면 |
| 강점 | 예측 정확도 | **해석(계수)** |
| 결측·스케일 | 거의 무관 | **둘 다 처리 필요**([C3](C3_인코딩과_결측_스케일링.md)) |

회귀는 단순하지만 그 단순함 덕에 **"각 변수가 얼마나 기여하는지"가 계수로 투명하게** 보입니다. 그게 트랙2가 부스팅 대신 회귀를 쓰는 이유입니다 — 우리는 점수가 아니라 **설명**이 필요하니까.

## 8. 핵심 정리

- 회귀 = 점들에 **직선**을 긋기. `가격 = b₀ + b₁·변수 + ...`, **계수 b₁ = "변수 1 증가 시 가격 변화"** = 알고 싶은 숫자.
- **OLS(최소제곱)** = 잔차(실제−예측)의 **제곱 합**을 최소로 하는 직선. 공식으로 한 번에.
- 변수 여럿이면 각 계수 = **"다른 조건이 같을 때"** 의 효과 → [R1](R1_회귀의_목적_인과.md) 통제가 수식으로 일어남.
- **로그 가격**을 쓰는 이유: ① 치우친 분포를 펴고 ② 계수가 **퍼센트 해석**이 됨.
- **R²** = 설명한 변동 비율. 효과 추정이 목적이면 낮아도 OK, **너무 높으면 누수 의심**.
- 회귀의 **4가정**(선형·독립·등분산·정규). 깨지면 계수보다 **신뢰도(표준오차)** 가 흔들림 → R5에서 보강.
