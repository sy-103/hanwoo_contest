## 1. 이 분석은 무엇을 추정하는가 (가장 중요한 절)

### 1-1. 질문의 형태가 다르다

분류팀의 질문: "이 소의 등급은 무엇인가?" (맞히면 됨)
**회귀팀의 질문: "더위(기상)는 한우 가격을 얼마나 움직이는가?"** (효과의 크기·방향·신뢰도를 말해야 함)

| | 분류 (19회차) | 회귀 (이번) |
| --- | --- | --- |
| 산출물 | 등급 예측값 | **회귀계수 + 신뢰구간 + 해석 문장** |
| 평가 | Macro-F1 | R²·RMSE는 보조, **계수의 신뢰성**이 본체 |
| 성공 기준 | 점수가 높다 | "폭염 노출 1%p 증가 → 가격 N% 변화 (95% CI: a~b)"를 말할 수 있다 |

> **train/test에 대하여**: 회귀팀은 **트랙2(기상 영향 분석)**라서 제출 파일이 없다. 그래서 진짜 test(`test_hanwoo.csv`)를 예측할 필요가 없다 — 게다가 test에는 COST_AMT(가격)가 아예 없어 평가할 수도 없다. 대신 우리는 분류팀과 **동일한 dev/holdout 분할**(split_assignment.csv)을 그대로 써서, 가격이 있는 train 데이터로 계수를 추정하고 그 계수가 **fold마다 안정적인지(5-fold CV)**, **홀드아웃에서도 유지되는지**로 신뢰성을 점검한다. 즉 회귀의 "test"는 제출용 파일이 아니라 **CV·홀드아웃을 통한 계수의 강건성 검증**이다. (만약 대회가 가격 예측 제출까지 요구한다면, 분류 강의안 3.5장의 `build_features_for_test`로 test 피처를 만든 뒤 학습한 회귀식에 그대로 대입하면 된다 — 단 가격은 log로 추정했으니 `np.exp()`로 되돌릴 것.)

### 1-2. 매개변수(mediator)는 넣으면 안 된다 — 기존 강의안에 없던 핵심 규칙

우리가 추정하려는 것은 **더위 → 가격의 총효과**다. 그런데 더위는 여러 단계를 **거쳐서** 가격에 영향을 준다:

```
더위 ──→ (사료섭취↓·스트레스) ──→ 도체중↓·일당증체(daily_gain)↓ ──→ 등급↓ ──→ 가격↓
   └──────────────────────────── 그 밖의 직접 효과 ──────────────────────────────↗
```

여기서 **이 화살표 사슬 위에 있는 변수**(등급·육질·**도체중 WEIGHT·일당증체 daily_gain**)를 우변에 넣으면, 모델은 "그 값이 같은 소들끼리" 비교하게 되어 그 변수를 거쳐 가는 효과가 전부 차단된다. 더위 효과가 실제보다 작게 나온다(매개변수 통제 편향).

> **비유**: "공부시간이 성적표 평균에 주는 효과"를 추정하면서 "중간고사 점수"를 통제하면 — 공부시간 효과 대부분이 중간고사를 통해 일어나므로 — 공부시간 효과가 0처럼 보인다.

**핵심 — WEIGHT·daily_gain도 매개변수다 (트랙1 분류와 정반대 취급).**
분류(19회차)에서는 WEIGHT·daily_gain이 "test에서 계산 가능한 합법 피처"라 적극 사용했다. **목적이 "예측"이라 무엇을 통해 맞히든 상관없기 때문**이다. 하지만 회귀의 목적은 "더위의 효과 추정"이고, 더위는 **체중·증체를 떨어뜨려서** 가격에 영향을 준다. 즉 WEIGHT·daily_gain은 더위의 *결과*(도축 후 측정값)이지 더위와 무관한 공통원인이 아니다 → **교란변수가 아니라 매개변수** → 총효과 모델 우변에서 제외.

> 실제로 이 데이터로 확인하면, WEIGHT·daily_gain을 우변에 넣는 순간 ratio_고온(더위) 계수가 약 **22% 작아진다**(매개를 통한 간접효과가 잘려나가기 때문). "교란 통제"라고 부르며 넣으면 더위 효과를 과소평가하게 된다.

**규칙**: 등급·육질·WGRADE·**WEIGHT·daily_gain**은 총효과 회귀의 우변 금지. ("이것들이 같을 때 더위의 직접효과"가 궁금하면 그건 *다른 질문*이고, 1-3절의 "직접효과 모델"로 따로 본다.)

### 1-3. 총효과 vs 직접효과 — 두 모델을 다 본다

매개변수를 빼면 **총효과**, 넣으면 **직접효과**다. 우리는 둘을 비교해서 그 차이를 해석한다.

| 모델 | 매개변수(WEIGHT·daily_gain) | 추정하는 것 | 역할 |
| --- | --- | --- | --- |
| **총효과 모델 (메인)** | 제외 | 더위가 가격에 주는 **전체** 영향 | 보고서 핵심 숫자 |
| **직접효과 모델 (보조)** | 포함 | "체중·증체가 같을 때" 더위의 **직접** 영향 | 비교용 |
| (총효과 − 직접효과) | — | 더위가 **체중·증체를 통해** 주는 **간접**효과 | 해석 스토리 |

> 단, 등급·육질은 **두 모델 모두에서 제외**한다(이건 test에도 없는 정답지라 누수이기도 함). 직접효과 모델이 추가로 통제하는 건 체격(WEIGHT·daily_gain)까지다.

**AGE(월령)는 경계선이다**: 도축 월령은 농가의 *출하 시기 선택*에 가까워(더위가 직접 정하는 값이 아님) 교란으로 통제해도 비교적 무방하다. 하지만 더위가 출하 시기를 앞당기는 경로가 있다면 약하게 매개일 수도 있으니, 민감도 점검(5장)에서 AGE 제외 버전도 한 번 본다.

### 1-4. 교란변수(confounder)는 반드시 넣어야 한다 — EDA가 준 답

15회차에서 "더위 많이 겪은 소가 등급이 좋다"는 반직관 결과가 나왔고, 그 원인으로 **지역 교란이 강하게 의심**됐다(광주 더움·등급낮음 9.18 / 인천 선선·등급높음 11.13 — 시도 단위에서 더위·등급 관계가 뒤집힘). 다만 **"확인"은 EDA가 아니라 바로 이 20회차 회귀가 한다** — 지역을 통제했을 때 더위 계수가 어떻게 변하는지가 그 검증이다. 16회차에서는 시즌성(추석 9월 피크)과 연도 차이(2025년 평균가 +12.8%)도 관찰됐다.

→ 그래서 회귀에는 **시도 더미, 계절 더미, 연도 더미**를 반드시 함께 넣는다. 이게 들어가야 "같은 지역·같은 시기 안에서" 더위가 다른 소들을 비교하게 되어, 비로소 순수한 날씨 효과에 가까워진다.

| 구분 | 예 | 처리 | 이유 |
| --- | --- | --- | --- |
| 교란 (더위와 무관하게 둘 다의 원인) | 지역·계절·연도·성별·농장규모 | **통제 (우변에 넣음)** | 안 넣으면 더위 효과가 오염됨 |
| 매개 (더위의 결과면서 가격의 원인) | 등급·육질·**WEIGHT·daily_gain** | **제외(총효과)** | 넣으면 더위 효과가 잘림 |
| 결과의 다른 표현 | WGRADE | 제외 | 같은 이유 |

### 1-4. 선택 편향 주의 — 가격은 63%만 있다

COST_AMT는 37%가 결측(원본 -99: 비공개 거래)이다. 회귀는 **가격이 공개된 소들만의 이야기**이며, 공개 여부가 무작위가 아닐 수 있다(특정 유통 경로). 이 한계는 보고서에 반드시 한 줄 명시한다.

## 2. 이론 — 공식과 함께

### 2-1. OLS (최소제곱법)

```
모형:  y = Xβ + ε
추정:  β̂ = (XᵀX)⁻¹ Xᵀy        — 잔차 제곱합 Σ(yᵢ − xᵢᵀβ)² 을 최소화하는 해
```

4대 가정(선형성·독립성·등분산성·정규성)의 정밀 진단은 21회차에서 하고, 이번 회차는 **이분산에 미리 대비**한다(아래 2-3).

### 2-2. 로그 종속변수와 퍼센트 해석 (정확한 공식)

가격은 오른쪽으로 길게 치우친 분포라 로그를 취한다. `log_cost = ln(COST_AMT)`.

```
모형:  ln(y) = β₀ + β₁x₁ + ... + ε

x₁이 1단위 증가할 때 y의 변화율:
  정확:  ( e^β₁ − 1 ) × 100 %        ← 보고서에는 이걸 쓴다
  근사:  β₁ × 100 %                  ← |β₁| < 0.1 일 때만 거의 같음
```

예: β = −0.008 → (e^(−0.008)−1)×100 = **−0.797%** (근사 −0.8%와 거의 같음).
β = 0.40 → 정확 +49.2% vs 근사 +40% — **계수가 크면 근사가 크게 틀린다.** 특히 더미 변수(예: 거세 vs 암)는 계수가 크기 쉬우므로 반드시 정확식으로.

### 2-3. 강건(robust) 표준오차 — HC1

가격의 분산은 소·농장마다 다를 수 있다(이분산). 이분산이 있으면 OLS의 **계수는 그대로 맞지만 표준오차(→p값·신뢰구간)가 틀린다.** 해결: 이분산에 강건한 HC1 표준오차.

```python
sm.OLS(y, X).fit(cov_type="HC1")        # 계수 동일, 표준오차만 강건하게
```

한 단계 더: 같은 농장의 소들은 오차가 서로 닮았을 수 있다(클러스터). 농장 클러스터 표준오차가 가장 보수적이다:

```python
sm.OLS(y, X).fit(cov_type="cluster", cov_kwds={"groups": df_reg["FARM_UNIQUE_NO"]})
```

본 강의안은 HC1을 기본으로, 클러스터를 점검용 옵션 셀로 제공한다. **둘의 결론(유의성)이 같으면 결과가 견고하다는 추가 증거**가 된다.

### 2-4. VIF — 다중공선성의 정량 측정

```
VIFⱼ = 1 / (1 − R²ⱼ)      R²ⱼ: 변수 j를 나머지 변수들로 회귀했을 때의 설명력
```

| VIF | 판정 |
| --- | --- |
| < 5 | 정상 |
| 5~10 | 의심 |
| > 10 | 심각 — 한쪽 제거 또는 규제 회귀 |

**계산 요령(중요)**: statsmodels의 `variance_inflation_factor`는 변수마다 회귀를 따로 돌려 122만 행 × 수십 변수에서 매우 느리다. 수학적으로 동일한 지름길이 있다 — **표준화된 변수들의 상관행렬 R의 역행렬 대각원소가 곧 VIF**다:

```
VIFⱼ = [R⁻¹]ⱼⱼ
```

아래 코드의 `fast_vif()`가 이것이며, 실데이터로 검증됐다.

**실제 검증에서 나온 사례 (미리 알려줌)**: AGE(월령)와 fattening_days(사육일수)의 VIF가 **6,400을 넘는다**. 당연하다 — 월령 ≈ 사육일수/30.4로 사실상 같은 변수다. → **둘 중 하나만 쓴다** (본 강의안은 도메인 표준인 AGE 채택). 이런 걸 걸러내는 게 VIF의 역할이다.

### 2-5. 규제 회귀 3종 — 목적함수

```
OLS:        min  Σ(yᵢ − xᵢᵀβ)²
Ridge:      min  Σ(yᵢ − xᵢᵀβ)²  +  α Σ βⱼ²            (L2 — 계수를 고르게 축소)
Lasso:      min  Σ(yᵢ − xᵢᵀβ)²  +  α Σ |βⱼ|           (L1 — 일부 계수를 정확히 0으로)
ElasticNet: min  Σ(yᵢ − xᵢᵀβ)²  +  α [ ρΣ|βⱼ| + (1−ρ)/2 Σβⱼ² ]   (ρ = l1_ratio)
```

- **Lasso**는 절댓값 페널티의 모서리 때문에 계수가 정확히 0이 된다 → **자동 변수 선택**.
- **Ridge**는 0을 만들지 않고 전체적으로 축소 → 다중공선성 완화에 유리.
- **ElasticNet**은 절충 — 상관 높은 변수 그룹에서 Lasso가 하나만 남기고 다 버리는 문제를 보완.
- α(규제 강도)가 클수록 계수가 더 눌린다. **규제 회귀는 변수 스케일에 민감 → 표준화 필수**.
- 규제 계수는 의도적으로 편향(축소)되어 있으므로 **p값·신뢰구간 해석은 OLS에서만** 한다. 규제 모델의 역할은 변수 선택과 예측 안정성 확인이다.

### 2-6. 두 가지 계수, 두 가지 용도

| | 어디서 얻나 | 용도 |
| --- | --- | --- |
| **원 단위 계수 β** | 원본 스케일 OLS | "x 1단위 ↑ → 가격 (e^β−1)×100% 변화" — **한계효과 보고** |
| **1-SD 효과 β·σₓ** | 위의 β에 x의 표준편차를 곱함 | 단위가 다른 변수끼리 **영향력 크기 비교** |

기존 강의안은 표준화 OLS를 돌린 뒤 σ로 나눠 원 계수를 복원했는데, 수학적으로 같지만 **0/1 더미까지 표준화하면 "1단위"의 의미가 깨진다.** 본 강의안은 원본 스케일로 한 번만 적합하고 두 해석을 모두 뽑는다 — 더 단순하고 더 정확하다.

### 셀 5 (코드) — 방어적 전처리 (실데이터 검증된 패턴)

```python
# ==============================================================
# 결측·무한대 방어 처리
# ==============================================================
def prepare_X(train_X, *other_Xs):
    """① inf→NaN ② train 중앙값으로 대치(누수 방지) ③ 전부결측 컬럼 제거"""
    train_X = train_X.replace([np.inf, -np.inf], np.nan)
    med = train_X.median(numeric_only=True)
    out = [train_X.fillna(med)]
    for X_ in other_Xs:
        out.append(X_.replace([np.inf, -np.inf], np.nan).fillna(med))
    bad = out[0].columns[out[0].isnull().any()].tolist()
    if bad:
        print(f"  [방어] 전부 결측 컬럼 제거: {bad}")
        out = [o.drop(columns=bad) for o in out]
    return out if len(out) > 1 else out[0]

X_full = prepare_X(df_reg[reg_features])
y_full = df_reg["log_cost"]
print(f"X: {X_full.shape}")
```

(중앙값 대치는 분포 중심을 보존하는 단순·안전한 선택이다. "기록 없음 자체가 정보"인 death_count류는 결측 더미를 추가하는 개선안이 5장에 있다.)

### 셀 6 (코드) — VIF 점검 (고속 계산 + 반복 제거)

```python
# ==============================================================
# 고속 VIF: 상관행렬 역행렬의 대각 = VIF  (실데이터 검증 완료)
# ==============================================================
def fast_vif(X):
    zero_var = X.columns[X.std() == 0].tolist()   # 분산 0이면 상관 정의 불가 → 방어
    Xv = X.drop(columns=zero_var)
    corr = Xv.corr().values
    vif = pd.Series(np.diag(np.linalg.pinv(corr)), index=Xv.columns)
    return vif.sort_values(ascending=False), zero_var

VIF_LIMIT = 10
removed = []
while True:
    vif, zv = fast_vif(X_full)
    removed += zv
    X_full = X_full.drop(columns=zv) if zv else X_full
    if vif.iloc[0] <= VIF_LIMIT:
        break
    worst = vif.index[0]
    print(f"제거: {worst} (VIF={vif.iloc[0]:.1f})")
    X_full = X_full.drop(columns=[worst]); removed.append(worst)

print(f"\n최종 변수 {X_full.shape[1]}개 (제거: {removed if removed else '없음'})")
print("VIF 상위 10:")
print(vif.head(10).round(2).to_string())
```

**읽는 법**: 제거 로그에 무엇이 나오는지 보라. AGE/fattening_days처럼 "정의상 같은 변수"가 걸리면 정상 작동이다. 관심 변수(ratio_고온 등)가 제거되면 어떤 통제 변수와 얽혔는지 13회차 상관표와 대조하라.

### 셀 10 (코드) — 핵심 기상 변수 한계효과 + 시나리오

```python
# ==============================================================
# 기상 변수 한계효과 — 보고서의 핵심 문장 재료
# ==============================================================
weather_vars = [v for v in ["ratio_고온","ta_min_mean","rn_day_mean",
                            "ws_davg_mean","heat_high"] if v in coef.index]
print("=== 기상 변수 효과 ===")
for v in weather_vars:
    b, p = coef.loc[v, "계수β"], coef.loc[v, "p"]
    lo_, hi_ = coef.loc[v, "CI하한"], coef.loc[v, "CI상한"]
    if v == "ratio_고온":   # 비율(0~1) → 0.01(=1%p) 단위가 자연스러움
        eff   = (np.exp(b*0.01)-1)*100
        eff_l = (np.exp(lo_*0.01)-1)*100; eff_h = (np.exp(hi_*0.01)-1)*100
        print(f"{v}: 생애 더위노출 1%p ↑ → 가격 {eff:+.3f}% "
              f"(95% CI {eff_l:+.3f}~{eff_h:+.3f}) p={p:.3g}")
    elif v == "heat_high":  # 더미 → 기준(하위 75%) 대비
        eff = (np.exp(b)-1)*100
        print(f"{v}: 고온노출 상위25% 소는 그 외 대비 가격 {eff:+.2f}% p={p:.3g}")
    else:
        eff = (np.exp(b)-1)*100
        print(f"{v}: 1단위 ↑ → 가격 {eff:+.3f}% p={p:.3g}")

# 마리당 금액 환산 시나리오
avg_price  = df_reg["COST_AMT"].mean()
avg_weight = df_reg["WEIGHT"].mean()
b = coef.loc["ratio_고온", "계수β"]
for dpp in [0.01, 0.05]:        # 더위노출 +1%p, +5%p 시나리오
    pct = (np.exp(b*dpp)-1)*100
    won = avg_price * pct/100 * avg_weight
    print(f"\n시나리오: 생애 더위노출 +{dpp*100:.0f}%p → 가격 {pct:+.2f}% "
          f"→ 마리당 약 {won:+,.0f}원 (평균가 {avg_price:,.0f}원/kg × {avg_weight:.0f}kg)")
```

**해석 작성 공식**: "[변수]가 [현실적인 단위]만큼 변하면, 같은 지역·같은 계절·같은 연도·같은 성별·같은 월령의 소 대비 가격이 약 N% (95% CI: a~b) 변한다" — **"같은 ~ 대비"(통제 조건)를 빼먹으면 인과처럼 들리는 과장 문장이 된다.**

### 셀 10.5 (코드) — 총효과 vs 직접효과 (매개 분해) ★

```python
# ==============================================================
# 매개(WEIGHT·daily_gain) 포함/제외 두 모델 비교 → 간접효과 추출
#   총효과(메인) = 위 ols / 직접효과 = 체격까지 통제한 모델
# ==============================================================
X_direct = prepare_X(df_reg[reg_features_direct])
ols_direct = sm.OLS(y_full, sm.add_constant(X_direct)).fit(cov_type="HC1")

b_total  = ols.params["ratio_고온"]            # 매개 제외 (총효과)
b_direct = ols_direct.params["ratio_고온"]     # 매개 포함 (직접효과)
eff_total  = (np.exp(b_total  * 0.01) - 1) * 100
eff_direct = (np.exp(b_direct * 0.01) - 1) * 100

print(f"총효과   (체격 미통제): 더위 1%p ↑ → 가격 {eff_total:+.3f}%  (R²={ols.rsquared:.3f})")
print(f"직접효과 (체격 통제)  : 더위 1%p ↑ → 가격 {eff_direct:+.3f}%  (R²={ols_direct.rsquared:.3f})")
print(f"→ 간접효과(더위가 체중·증체를 통해): 약 {eff_total - eff_direct:+.3f}%p")
print("   해석: 더위 효과 중 이만큼이 '체중·증체를 떨어뜨려서' 가격에 작용한 몫.")
```

**왜 이 셀이 중요한가**: WEIGHT·daily_gain은 더위의 *결과*(매개변수)이므로, 메인 보고 숫자는 **총효과(이들을 뺀 모델, = 위 ols)** 를 쓴다. 직접효과 모델은 "체중이 같을 때의 잔여 효과"라는 *다른 질문*의 답이며, 둘의 차이가 "더위 → 체중·증체 → 가격"의 간접 경로 크기다. **보고서에는 총효과를 헤드라인으로, 간접효과 분해를 부연으로** 쓴다.

### 셀 12 (코드) — 표준화 + 규제 회귀 3종 (5-fold CV)

```python
# ==============================================================
# Lasso / Ridge / ElasticNet — 표준화 후, 분류팀과 같은 fold로 CV
# ==============================================================
LASSO_ALPHA, RIDGE_ALPHA = 0.001, 1.0      # TODO: 5장 가이드로 탐색
EN_ALPHA, EN_L1 = 0.001, 0.5

folds = [(np.where(df_reg["fold"] != k)[0], np.where(df_reg["fold"] == k)[0])
         for k in range(5)]
makers = {
    "OLS":        lambda: LinearRegression(),
    "Lasso":      lambda: Lasso(alpha=LASSO_ALPHA, max_iter=10000, random_state=SEED),
    "Ridge":      lambda: Ridge(alpha=RIDGE_ALPHA, random_state=SEED),
    "ElasticNet": lambda: ElasticNet(alpha=EN_ALPHA, l1_ratio=EN_L1,
                                     max_iter=10000, random_state=SEED),
}
cv = {m: {"r2": [], "rmse": []} for m in makers}
nz_last = {}
for tr_idx, va_idx in folds:
    X_tr, X_va = prepare_X(df_reg[reg_features].iloc[tr_idx],
                           df_reg[reg_features].iloc[va_idx])
    y_tr, y_va = y_full.iloc[tr_idx], y_full.iloc[va_idx]
    sc = StandardScaler()
    X_tr_s = sc.fit_transform(X_tr); X_va_s = sc.transform(X_va)   # train으로만 fit
    for name, mk in makers.items():
        m = mk().fit(X_tr_s, y_tr)
        pred = m.predict(X_va_s)
        cv[name]["r2"].append(r2_score(y_va, pred))
        cv[name]["rmse"].append(np.sqrt(mean_squared_error(y_va, pred)))
        if hasattr(m, "coef_"):
            nz_last[name] = int((np.abs(m.coef_) > 1e-10).sum())

cv_summary = pd.DataFrame({
    "모델": list(makers),
    "R²평균":   [np.mean(cv[m]["r2"]) for m in makers],
    "R²표준편차":[np.std(cv[m]["r2"])  for m in makers],
    "RMSE평균": [np.mean(cv[m]["rmse"]) for m in makers],
    "비영(≠0)계수": [nz_last.get(m, X_tr.shape[1]) for m in makers],
}).round(4)
print(cv_summary.to_string(index=False))
cv_summary.to_csv("../../data/processed/4_model/20_cv_results.csv",
                  index=False, encoding="utf-8-sig")
```

**파라미터 설명**
| 파라미터 | 의미 | 주의 |
| --- | --- | --- |
| `alpha` | 규제 강도. 클수록 계수가 더 눌리고(Lasso는 더 많이 0) R²는 보통 내려감 | 표준화된 X 기준 값. 0.0001~1을 로그 간격으로 탐색 |
| `l1_ratio` | ElasticNet의 L1 비중 (0=Ridge, 1=Lasso) | 0.2/0.5/0.8 정도부터 |
| `max_iter=10000` | 좌표하강 수렴 한도 — 기본값(1000)은 이 규모에서 수렴 경고가 흔함 | 경고가 나면 alpha를 키우거나 iter 증가 |

> **OLS가 두 군데 나온다 — 변수셋이 다름에 주의**: 셀 7~8의 **해석용 OLS**는 VIF로 공선성 변수를 제거한 `X_full`을 쓰고, 셀 12의 **비교용 OLS**(sklearn)는 VIF 제거 없이 `reg_features` 전체를 쓴다(규제 모델은 공선성을 알아서 처리하므로 전체로 두는 게 합리적). 그래서 두 OLS의 R²가 미세하게 다를 수 있는데 정상이다. **계수 해석은 항상 셀 8(해석용 OLS), 예측 성능 비교는 셀 12**로 역할을 분리해 읽는다.

## 4. 결과 해석 가이드 — 계수표를 받아들고 읽는 순서

> 숫자는 직접 돌려서 얻는다. 아래는 "읽는 절차"다.

**① 관심 변수(기상)부터, 다섯 단계로:**
1. **부호** — 도메인 예상(더위↑→가격↓?)과 맞는가. 15회차에서 본 단순 상관은 +0.30(교란 오염)이었다. 지역·계절 통제 후 부호가 어떻게 되는지가 이 분석의 하이라이트다. 통제 후에도 +라면 "더위가 좋다"가 아니라 남은 교란(농가 수준 등)을 먼저 의심하라.
2. **크기** — (e^β−1)×100%를 현실 단위로. "ratio_고온 1%p ↑ → N%"가 0.01% 수준이면 통계적으로 유의해도 실무적으론 무시 가능.
3. **신뢰구간** — CI가 0을 포함하면 방향조차 단정 못 한다.
4. **VIF** — 그 변수의 VIF가 높았다면 계수 해석을 보류.
5. **강건성** — HC1과 클러스터 SE에서 결론이 같은가.

**② p값은 122만 행에선 거의 다 작다.** 유의성(별 개수)이 아니라 **효과 크기(%)와 CI 폭**으로 이야기하라. "p<0.001이므로 중요하다"는 이 표본 크기에서 공허한 문장이다.

**③ R² 읽는 법** — log 가격 분산의 설명 비율. 개체 단위 가격에는 모델 밖 요인(경매 당일 수급 등)이 많아 R²가 낮아도 계수 해석의 가치는 별개다. R²가 예상 밖으로 매우 높다면 오히려 누수(매개변수 유입)를 의심하라.

**④ 더미 계수는 "기준 대비"다.** sido 더미의 기준은 강원(drop_first), 계절은 가을, 성별은 거세. "sido_제주 = −0.1"은 "강원 대비 제주가 (e^(−0.1)−1) ≈ −9.5%"라는 뜻이다 — 기준을 안 밝히면 틀린 문장이 된다.

**⑤ OLS vs 규제 모델 비교** — 네 모델의 CV R²가 비슷하면 "결과가 모델 선택에 둔감하다"는 강건성 증거. Lasso가 변수를 많이 버려도 R²가 유지되면 그 소수 변수가 정보의 본체다.

**⑥ 분류팀의 SHAP과 교차 검증** — 회귀의 1-SD 영향력 상위와 분류의 SHAP 상위가 비슷하게 나오는지 비교하라. 두 트랙이 같은 변수를 가리키면 보고서의 설득력이 크게 올라간다.

## 7. 총정리

1. **회귀의 산출물은 점수가 아니라 문장**이다: "같은 지역·계절·연도·성별·월령 조건에서, 생애 더위노출 1%p 증가는 가격을 N% (CI a~b) 움직인다."
2. **설계가 절반**: 교란(지역·계절·연도 등)은 통제, 매개(등급·육질·**체중·증체**)는 제외 = 총효과. 매개를 넣은 직접효과 모델과 비교해 간접경로를 분해한다. 15회차 U자형 교란의 해소 장치가 바로 시도 더미다.
3. **공선성은 VIF로 정량 처리**: AGE↔사육일수(VIF 6,400+) 같은 중복은 하나만. 고속 VIF(상관행렬 역행렬)로 122만 행에서도 즉시 계산.
4. **불확실성은 강건하게**: HC1 기본 + 농장 클러스터 SE로 재확인. 두 결론이 일치하면 견고.
5. **해석은 정확식으로**: (e^β−1)×100%, 더미는 기준 카테고리 명시, 영향력 비교는 1-SD 효과.
6. **OLS가 해석의 본체, 규제 3종은 변수 선택과 강건성의 증인**이다.
7. 향상의 지렛대는 파라미터가 아니라 **사양**(비선형·상호작용·90일 윈도우·고정효과)과 **민감도 점검**이다.
