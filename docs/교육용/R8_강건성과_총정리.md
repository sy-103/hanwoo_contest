# R8. 강건성 점검과 총정리 (회귀)

> **이 강의의 목표**: 회귀로 얻은 결과를 **믿어도 되는지** 확인하는 법(강건성·민감도 점검)을 배우고, 점수를 더 정교하게 만드는 방향과, 회귀 트랙 전체를 총정리합니다.
> **앞 강의**: R1~R6 전부.

## 1. 왜 "강건성"을 점검하나

[R1](R1_회귀의_목적_인과.md)에서 회귀의 목적은 "더위 효과를 **믿을 수 있게** 말하는 것"이라 했죠. 그런데 계수 하나 뽑고 끝내면 위험합니다 — 그 숫자가 **우연이거나, 특정 가정·표본에만 의존**할 수 있으니까요.

**강건성(robustness) 점검** = "방법을 조금 바꿔도 결론이 그대로인가?"를 확인하는 것. 결론이 흔들리면 그 결과는 약한 거고, 여러 방법에서 똑같이 나오면 믿을 만한 겁니다.

> 비유: 체중계가 정확한지 보려면, 다른 체중계로도 재보고, 다른 시간에도 재봅니다. 다 비슷하면 믿고, 들쭉날쭉하면 의심하죠. 회귀 결과도 똑같이 "여러 각도로 재봅니다."

---

## 2. 점검 ① 표준오차를 바꿔본다 (이미 R5에서)

[R5](R5_계수해석_한계효과.md)에서 배운 것. **HC1(이분산 강건)** 과 **농장 클러스터** 표준오차로 신뢰도를 재봅니다.

- 두 방법에서 **유의성 결론이 같으면** → 견고.
- 클러스터에서만 유의성이 사라지면 → "농장 수준 현상일 수 있다"고 솔직히 보고.

## 3. 점검 ② 모델을 바꿔본다 (R4 규제)

[R4](R4_규제회귀_라쏘릿지.md)의 규제 회귀가 여기서 쓰입니다. OLS·Lasso·Ridge·ElasticNet의 결과를 비교:

- 네 모델의 **예측력(R²)이 비슷**하면 → "결과가 모델 선택에 둔감하다"는 증거.
- **Lasso에서도 더위 변수가 살아남으면**(0이 안 되면) → "규제를 걸어도 버틸 만큼 독립적인 정보"라는 증거.

규제 모델이 더위 변수를 버리거나 OLS와 부호가 다르면, 그 효과는 약하거나 다른 변수에 의존적이라는 신호입니다.

---

## 4. 점검 ③ 표본을 바꿔본다 (민감도 분석)

같은 모델을 **다른 부분집합**으로 돌려, 결론이 유지되는지 봅니다. 우리 데이터의 특성에 맞춘 점검들:

| 점검 | 무엇을 | 왜 |
| --- | --- | --- |
| **노폐우 제외** | 2016년 이후 출생만 | [16회차]에서 본 이질 집단(번식 끝난 늙은 소) 빼고도 같나 |
| **연도별 분리** | 2023/2024/2025 각각 | 효과가 시기마다 안정적인가 |
| **거세우만** | 가장 동질적 집단 | 성별 차이를 제거한 깨끗한 효과 |
| **AGE 제외** | 월령을 빼고 | AGE가 약한 매개일 수도 있으니([R1](R1_회귀의_목적_인과.md)) 빼도 더위 효과가 같나 |

결과가 부분집합마다 크게 출렁이면, 그 자체를 **보고서에 정직하게 기록**합니다. "어떤 집단에선 효과가 크고 어떤 집단에선 작다"도 훌륭한 발견입니다.

## 5. 결과를 더 정교하게 — 사양(specification) 개선

회귀에서 점수(설명력·정밀도)를 올리는 건 분류의 파라미터 튜닝과 다릅니다. **"모델 사양"** 을 바꾸는 게 핵심입니다.

| 시도 | 방법 | 효과 |
| --- | --- | --- |
| **비선형 항** | 월령² 같은 제곱항 추가 | [14회차] LOWESS가 본 곡선 관계 포착 |
| **상호작용 항** | 더위 × 밀집도 | "더위 피해가 어떤 조건에서 더 큰가" |
| **시점 세분화** | 도축 전 90일 더위 변수 신설 | "생애 전체"보다 "출하 직전 스트레스"가 가격에 민감할 수도 |
| **농장 고정효과** | 농장 평균을 빼고 회귀 | 농장 수준 교란을 통째로 제거(가장 깐깐) |
| **분위 회귀** | 저가/중간/고가 소 따로 | "더위 피해가 저가 소에서 더 큰가" |

> 분류는 "손잡이(하이퍼파라미터)"를 돌려 성능을 올리지만, 회귀는 **"어떤 변수를 어떤 형태로 넣느냐(사양)"** 가 더 큰 지렛대입니다. 데이터를 더 잘 표현하는 식을 찾는 거죠.

## 6. 회귀팀이 보고서에 꼭 담을 것

회귀의 산출물은 [R1](R1_회귀의_목적_인과.md)에서 말한 "문장"입니다. 보고서엔 이걸 담습니다.

1. **헤드라인 효과**: "더위 1%p↑ → 가격 N% ([95% CI])" — 총효과 기준([R5](R5_계수해석_한계효과.md)).
2. **영향력 순위**: 1-SD 효과로 어떤 변수가 가격을 크게 좌우하는지.
3. **간접 경로**: 총효과 vs 직접효과 분해(더위가 체격을 통해 주는 몫).
4. **강건성**: HC1/클러스터/규제/부분집합에서 결론이 유지됨.
5. **한계 명시 (정직성)**:
   - **선택 편향**: "가격 공개 거래(63%)에 한정한 결과"([R1](R1_회귀의_목적_인과.md)).
   - **관측 연구**: 측정 못 한 교란이 있을 수 있어 인과를 단정하지 않음.
   - 부호가 양수면 남은 지역 교란 가능성 언급.

> **정직성이 점수입니다.** 트랙2는 정성 평가(합리성·분석력)라, "이런 한계가 있다"를 스스로 밝히는 게 오히려 신뢰를 높입니다.

## 7. 회귀 트랙 총정리 (R1~R8 한눈에)

```
[R1] 회귀의 목적 = 설명. 상관≠인과, 교란(넣기)·매개(빼기)
  ↓
[R2] 선형회귀 = 잔차 제곱 최소(OLS). 로그로 %해석. 평가지표(RMSE·MAE·R²). 4가정
  ↓
[R3] 다중공선성 = 변수 닮음 → 계수 불안정. VIF로 진단(>10 제거)
  ↓
[R4] 규제 = 계수에 벌금. Lasso(변수선택)·Ridge(축소)·ElasticNet
  ↓
[R5] 계수 해석 = (e^b−1)×100%, 1-SD 비교, 더미는 기준대비, CI·강건SE
  ↓
[R6] 코드로 실행
  ↓
[R7] 잔차 진단 = 4가정 검사(잔차 산점도·Q-Q·쿡의 거리)
  ↓
[R8] 강건성(모델·표본 바꿔도 같나) + 사양 개선 + 정직한 한계
```

**관통하는 한 가지**: 회귀는 처음부터 끝까지 **"이 효과를 믿어도 되나?"** 를 묻습니다. 교란을 통제하고(R1), 다중공선성을 걷어내고(R3), 강건 표준오차로 신뢰도를 보강하고(R5), 가정을 검진하고(R7), 여러 방법으로 재확인하는(R8) 모든 것이 **"믿을 수 있는 숫자 하나"** 를 만들기 위한 과정입니다.

## 8. 분류 트랙과의 마지막 비교

두 트랙을 다 본 지금, 차이를 음미해 봅시다.

| | 분류 (트랙1) | 회귀 (트랙2) |
| --- | --- | --- |
| 목표 | 잘 **맞히기** | 잘 **설명하기** |
| 핵심 가치 | 일반화(처음 보는 데이터 점수) | 신뢰성(믿을 수 있는 효과) |
| 몸무게·증체 | 적극 사용(예측에 도움) | 매개라 제외(총효과) |
| 평가 | Macro-F1 점수 | 계수·신뢰구간·강건성 |
| 함정 | 누수·불균형 | 교란·매개·다중공선성 |
| 모델 | 부스팅(블랙박스, 강력) | 선형(투명, 해석) |

**같은 데이터, 다른 질문, 다른 도구.** 어떤 질문을 하느냐가 어떤 방법을 쓸지를 정한다는 것 — 이게 데이터 분석의 가장 중요한 출발점입니다.

---

## 9. 핵심 정리

- **강건성 점검** = 방법을 바꿔도 결론이 같은지 확인. ① 표준오차(HC1/클러스터) ② 모델(규제) ③ 표본(노폐우·연도·성별·AGE 제외).
- 회귀 성능의 지렛대는 파라미터가 아니라 **사양**(비선형·상호작용·90일 윈도우·고정효과·분위).
- 보고서엔 **헤드라인 효과 + 영향력 순위 + 간접경로 + 강건성 + 정직한 한계**(선택편향·인과 단정 금지).
- 회귀 트랙을 관통하는 한 가지: **"이 효과를 믿어도 되나?"**
- 분류 vs 회귀: **같은 데이터, 다른 질문, 다른 도구.** 질문이 방법을 정한다.
