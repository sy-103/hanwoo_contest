# R1. 회귀의 목적 — 예측이 아니라 "설명"

> **이 강의의 목표**: 회귀가 분류와 **목적이 어떻게 다른지**(맞히기 vs 설명하기), 그리고 회귀 분석의 심장인 **상관 ≠ 인과, 교란변수, 매개변수** 개념을 처음부터 배웁니다. 이게 흔들리면 회귀 결과를 통째로 잘못 해석하게 됩니다. 공식은 R2부터, 여기선 사고방식을 잡습니다.
> **앞 강의**: [00-B](00B_우리프로젝트_전체그림.md)의 트랙2·"상관≠인과" 맛보기. (분류의 [C1](C1_분류문제와_평가지표.md)~[C3](C3_인코딩과_결측_스케일링.md)도 읽으면 데이터 분할·인코딩 이해에 도움.)


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 회귀 1/8**

## 1. 회귀팀의 질문은 분류팀과 다르다

먼저 두 팀의 질문을 나란히 놓아봅시다.

- **분류팀**: "이 소의 등급은 무엇인가?" → **잘 맞히면 끝.** 무슨 변수를 쓰든 점수만 높으면 됨.
- **회귀팀**: "**더위**는 한우 **가격**을 얼마나 움직이나?" → **"얼마나, 어느 방향으로, 믿을 만한가"를 설명**해야 함.

이 차이가 회귀 분석 전체를 지배합니다.

| | 분류 (트랙1) | 회귀 (트랙2) |
| --- | --- | --- |
| 진짜 목적 | **예측 정확도** | **효과의 해석** |
| 핵심 산출물 | 등급 예측값 | "더위 1%p↑ → 가격 N% (95% 구간 a~b)" 같은 **문장** |
| 무엇을 보나 | Macro-F1 점수 | 계수의 크기·방향·신뢰도 (R²는 보조) |
| 성공 | 점수가 높다 | 효과를 믿을 수 있게 말한다 |

> **핵심 한 줄**: 회귀의 산출물은 **점수가 아니라 문장**입니다. "더위가 가격을 이만큼 떨어뜨린다"를 **근거와 함께** 말하는 게 목표입니다.

이 때문에 회귀에선 **"어떤 변수를 넣느냐"가 결과의 의미를 바꿉니다.** 분류는 점수만 높으면 되니 막 넣어도 되지만, 회귀는 변수 하나를 잘못 넣으면 "더위 효과"라는 숫자 자체가 틀려집니다. 왜 그런지가 이 강의의 핵심입니다.

---

## 2. 회귀를 한 문장으로 + 예측용 회귀와의 구분

**회귀(regression)** = 숫자 타깃(가격)을, 여러 입력변수의 **관계식**으로 설명·예측하는 것. 가장 단순한 형태:

```
가격 = b₀ + b₁·더위 + b₂·몸무게 + b₃·월령 + ...
```

각 `b`(계수)가 "그 변수가 1 늘면 가격이 얼마 변하나"를 나타냅니다. **이 계수를 해석하는 게 회귀 분석의 전부**라 해도 됩니다.

> 주의: 회귀를 **"가격 예측기"** 로 쓸 수도 있습니다(분류처럼). 하지만 우리 트랙2의 목적은 예측이 아니라 **"더위의 효과 추정"** 입니다. 그래서 R²(예측 잘 맞나)보다 **계수가 믿을 만한가**가 중요합니다. 이 차이를 계속 기억하세요.

---

## 3. 가장 중요한 함정: 상관 ≠ 인과

회귀 분석에서 **가장 많이, 가장 크게 망하는** 지점입니다. 외우세요: **"상관이 있다고 원인인 건 아니다."**

### 고전 사례: 아이스크림과 익사

여름에 데이터를 보면 **아이스크림 판매량**과 **익사 사고**가 같이 늘어납니다(강한 양의 상관). 그럼 "아이스크림이 익사를 유발한다"? 당연히 아니죠. 숨은 제3의 변수 **여름(더위)** 이 둘 다 끌어올린 겁니다. 사람들이 더우니 아이스크림을 사고, 더우니 물에 들어가서 사고가 나는 거죠.

```
        여름(더위)  ← 숨은 진짜 원인
        /        \
아이스크림↑      익사↑
   (이 둘은 직접 인과관계가 없는데도 같이 움직임 = 가짜 상관)
```

이 **숨은 제3의 변수**를 **교란변수(confounder)** 라고 합니다.

### 우리 데이터의 진짜 사례

[00-B](00B_우리프로젝트_전체그림.md)에서 봤죠. EDA 결과 **"더위 많이 겪은 소가 등급이 좋다"** 가 나왔습니다. 상식과 반대입니다. 여기서 "더위가 좋다"고 결론 내리면 아이스크림-익사와 똑같은 실수입니다.

진짜 원인은? **지역**일 가능성이 큽니다. 더운 남부 지역에 *마침* 사육 기술 좋은 우량 농가가 몰려 있다면:

```
        더운 지역(남부)  ← 교란변수
        /            \
   더위 노출↑       우량 농가↑ → 등급↑
   
   "더위 → 등급↑"처럼 보이지만, 진짜는 "지역 → 둘 다"
```

실제로 시도 단위로 보면 광주(가장 더움)는 등급 9.18(낮음), 인천(선선)은 11.13(높음)으로 **"더우면 등급 높다"가 깨집니다.** 단순 상관에 속으면 안 된다는 증거죠.

---

## 4. 교란변수를 다루는 법: 통제(control)

그럼 어떻게 진짜 더위 효과를 알까요? **교란변수를 회귀에 같이 넣습니다.** 이걸 **"통제한다(control for)"** 고 합니다.

직관: 회귀에 지역을 같이 넣으면, 모델이 **"같은 지역 안에서" 더위가 다른 소들끼리** 비교하게 됩니다. 지역이라는 차이를 고정시켜 버리니, 남는 건 순수한 더위 효과에 가까워집니다.

```
지역을 안 넣으면:  더위 효과 = 진짜 더위 효과 + 지역의 가짜 효과 (오염됨)
지역을 넣으면:     더위 효과 ≈ 진짜 더위 효과 (지역 차이를 걷어냄)
```

> 그래서 우리 회귀엔 **시도·계절·연도·성별** 같은 교란 후보를 반드시 함께 넣습니다. 이게 R5에서 "통제 변수"로 다시 나옵니다. **"같은 지역·같은 계절·같은 성별 조건에서 더위가 가격에 주는 효과"** — 이 "같은 ~ 조건에서"가 통제의 결과입니다.

> 솔직한 한계: 통제는 **우리가 측정한 교란만** 걷어냅니다. 측정 못 한 숨은 교란이 있으면 여전히 오염될 수 있어요. 그래서 관측 데이터 분석은 "인과"를 **단정**하지 않고 "강하게 시사한다" 정도로 신중하게 말합니다.

---

## 5. 또 하나의 함정: 매개변수 (넣으면 안 되는 변수)

교란변수는 **넣어야** 합니다. 그런데 정반대로, **넣으면 안 되는** 변수도 있습니다. **매개변수(mediator)** 입니다. 이게 회귀팀이 가장 많이 헷갈리는 지점이라 천천히 봅니다.

### 매개변수란

매개변수는 **원인과 결과 사이의 "경유지"** 입니다. 더위가 가격에 영향을 주는 경로를 그려봅시다.

```
더위 → (사료 못 먹음·스트레스) → 몸무게↓·증체↓ → 등급↓ → 가격↓
```

여기서 **몸무게·증체·등급은 더위의 "결과"이면서 가격의 "원인"** 입니다. 더위가 이것들을 거쳐서 가격에 영향을 주는 거죠. 이런 중간 다리를 매개변수라 합니다.

### 왜 넣으면 안 되나

우리가 알고 싶은 건 **더위 → 가격의 전체(총) 효과** 입니다. 그런데 몸무게를 회귀에 넣으면? 모델이 **"몸무게가 같은 소들끼리"** 비교하게 됩니다. 그러면 "더위가 몸무게를 떨어뜨려서 가격을 낮추는" 경로가 **차단**됩니다. 더위 효과의 상당 부분이 사라져서, 실제보다 작게 나옵니다.

> 비유: **"공부시간이 성적에 주는 효과"** 를 알고 싶은데 **"중간고사 점수"** 를 통제하면? 공부의 효과 대부분이 중간고사를 거쳐 일어나므로, 공부시간 효과가 거의 0으로 보입니다. 중간고사는 공부의 *결과*(매개)이지, 통제할 교란이 아니니까요.

### 교란 vs 매개 — 결정적 구분

| | 교란변수 | 매개변수 |
| --- | --- | --- |
| 위치 | 원인·결과 **둘 다의 공통 원인** | 원인과 결과 **사이의 경유지** |
| 시간 | 더위와 **무관/이전** | 더위 **이후**에 결정 |
| 예 | 지역·계절·성별 | 등급·몸무게·증체 |
| 처리 | **넣는다(통제)** | **뺀다(총효과 기준)** |
| 안 지키면 | 효과가 오염됨 | 효과가 잘려나감 |

> **구분 질문**: *"이 변수는 더위보다 먼저 정해졌나(교란), 더위 때문에 변하나(매개)?"* 지역은 소가 태어나기 전부터 정해진 것(교란), 몸무게는 더위를 겪으며 변한 것(매개)입니다.

### 우리 데이터의 핵심 주의

여기서 **분류와 회귀가 정반대**가 됩니다.

- **분류(트랙1)**: 몸무게·daily_gain은 "test에서 계산 가능한 합법 피처"라 **적극 사용**([C2](C2_데이터분할과_누수.md)). 목적이 *예측*이라 무엇을 거쳐 맞히든 상관없으니까.
- **회귀(트랙2)**: 몸무게·daily_gain은 더위의 *결과(매개)* 라 **총효과 모델에서 제외**. 넣으면 더위 효과가 잘림.

실제로 우리 데이터로 확인하면, 몸무게·증체를 넣는 순간 더위 효과가 약 22% 작아집니다. 같은 변수가 트랙에 따라 운명이 갈리는 겁니다.

> 등급·육질은 두 트랙 모두에서 제외(매개이기도 하고 누수이기도 함). 회귀에서 추가로 빼는 건 **몸무게·daily_gain(체격)** 입니다.

### 그럼 매개는 영영 못 보나? — 총효과 vs 직접효과

매개를 **뺀** 모델 = **총효과**(더위의 전체 영향), 매개를 **넣은** 모델 = **직접효과**("체격이 같을 때" 더위의 잔여 영향). 둘을 **다 돌려서 비교**하면, 그 차이가 "더위가 체격을 통해 주는 **간접효과**"입니다. 우리는 총효과를 헤드라인으로, 간접효과 분해를 부연으로 보고합니다. (R5·R7에서 코드로.)

## 6. 선택 편향도 기억하기

마지막 함정. 우리 가격(COST_AMT)은 **37%가 비어 있습니다**(비공개 거래). 회귀는 **가격이 공개된 소들만** 봅니다. 만약 공개 여부가 무작위가 아니면(예: 특정 유통경로만 공개), 결과가 그 집단에만 해당하는 **선택 편향(selection bias)** 이 생깁니다. 그래서 보고서엔 *"가격 공개 거래(63%)에 한정한 결과"* 라는 한 줄을 꼭 답니다.

## 7. 핵심 정리

- 회귀의 목적은 **예측이 아니라 설명** — 산출물은 점수가 아니라 **"더위가 가격을 N% 움직인다"는 문장**.
- **상관 ≠ 인과.** 같이 움직인다고 원인이 아니다(아이스크림-익사).
- **교란변수** = 둘 다의 공통 원인(지역). **반드시 통제(회귀에 넣음)** → "같은 지역 안에서" 비교.
- **매개변수** = 원인-결과 사이 경유지(몸무게·등급). **총효과 모델에선 제외** → 안 그러면 효과가 잘림.
- 구분 질문: *"더위보다 먼저냐(교란), 더위 때문에 변하냐(매개)?"*
- 분류는 몸무게를 쓰고 회귀는 뺀다 — **목적이 다르기 때문.**
- **총효과(매개 제외) vs 직접효과(매개 포함)** 를 비교해 간접경로를 본다.
- 가격 37% 결측 → **선택 편향** 명시.
