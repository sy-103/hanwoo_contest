# C2. 데이터 나누기와 데이터 누수

> **이 강의의 목표**: 왜 데이터를 쪼개야 하는지(과적합 복습), 교차검증이 무엇이고 어떤 종류가 있는지, 우리가 왜 "농장 단위 + 등급 비율 유지" 분할을 쓰는지, 그리고 분석을 망치는 여러 종류의 "누수"를 빠짐없이 배웁니다. 마지막으로 불균형을 다루는 클래스 가중치까지.

## 1. 출발점: 왜 나누나 (과적합 다시 보기)

[00-A](00A_데이터분석이란.md)에서 모델의 진짜 목표는 **외우기가 아니라 일반화**(처음 보는 데이터를 잘 맞히기)라고 했습니다. 그런데 문제가 있습니다 — 모델이 **외웠는지**, **이해했는지**를 어떻게 구분하죠?

학습에 쓴 데이터로 채점하면 못 구분합니다. 외운 학생도 기출에선 100점이니까요. **유일한 방법은 "학습에 안 쓴 데이터"로 시험 보는 것**입니다. 그래서 데이터를 나눕니다. 이게 데이터 분할의 존재 이유 전부입니다.

```
가진 데이터 ─┬─ 학습용(train) : 모델이 공부하는 교재
             └─ 시험용(test)  : 안 보여준 문제로 채점
```

---

## 2. 3분할: 학습 / 검증 / 테스트

실제로는 두 조각이 아니라 **세 조각**이 필요합니다. 왜 그런지 단계적으로 봅시다.

처음엔 train/test 둘로 나눕니다. train으로 배우고 test로 채점. 그런데 우리는 모델을 **여러 번 고칩니다**(하이퍼파라미터를 바꿔가며). 매번 test로 채점해서 제일 좋은 걸 고르면? **test에 맞춰 튜닝**하게 되어, test가 사실상 "두 번째 학습 데이터"가 됩니다. test도 오염되는 거죠.

그래서 한 조각을 더 뗍니다.

| 조각 | 이름 | 역할 |
| --- | --- | --- |
| ① | **학습(train/dev)** | 모델이 공부 |
| ② | **검증(validation)** | 튜닝하면서 "이 설정이 더 나은가" 비교 — **여러 번 봄** |
| ③ | **테스트(test/holdout)** | 모든 게 끝난 뒤 **딱 한 번** 최종 채점 |

우리 프로젝트 용어로는: ①②가 **dev**(그 안에서 다시 학습/검증으로 나눔), ③이 **holdout**, 그리고 진짜 대회 **test**는 정답이 없는 별도 파일입니다([00-B](00B_우리프로젝트_전체그림.md) 참고).

> 비유: ① 평소 공부, ② 모의고사(여러 번 보며 약점 보완), ③ 수능(딱 한 번). 모의고사를 수능처럼 여러 번 보면 안 되는 이유와, 수능을 미리 보면 안 되는 이유가 정확히 ②③입니다.

## 3. 홀드아웃의 약점 → 교차검증의 등장

검증을 위해 dev를 그냥 80:20으로 한 번 자르는 방법을 **홀드아웃(hold-out)** 이라고 합니다. 간단하지만 약점이 있습니다.

- **운(運)에 휘둘립니다.** 우연히 쉬운 소들이 검증셋에 몰리면 점수가 뻥튀기, 어려운 소가 몰리면 저평가. **딱 한 번 자른 결과**라 믿음이 약합니다.
- 검증에 쓴 20%는 **학습에 못 씁니다.** 데이터가 아까워요.

이 두 문제를 한 번에 푸는 게 **교차검증(Cross-Validation, CV)** 입니다.

### K-겹 교차검증(K-Fold) 원리

데이터를 **K개 조각(fold)** 으로 나눕니다(보통 5개). 그리고 **돌아가면서** 한 조각씩 검증셋으로 씁니다.

```
데이터를 5조각으로:  [1][2][3][4][5]

1회차:  검증=[1]  학습=[2][3][4][5]   → 점수₁
2회차:  검증=[2]  학습=[1][3][4][5]   → 점수₂
3회차:  검증=[3]  학습=[1][2][4][5]   → 점수₃
4회차:  검증=[4]  학습=[1][2][3][5]   → 점수₄
5회차:  검증=[5]  학습=[1][2][3][4]   → 점수₅

최종 = 점수₁~₅의 평균 ± 표준편차
```

### 교차검증의 장점

1. **모든 데이터가 검증에 한 번씩** 쓰입니다(낭비 없음). 동시에 모든 데이터가 학습에도 4번씩 쓰입니다.
2. 점수가 **5개** 나오니 **평균(실력)과 표준편차(안정성)** 를 같이 봅니다. 표준편차가 크면 "이 모델은 데이터에 따라 들쭉날쭉"이라는 경고입니다.
3. 한 번 자른 운에 휘둘리지 않습니다.

> 우리는 **dev를 5-fold로** 교차검증합니다. holdout은 그와 별개로, 5-fold가 다 끝난 뒤 딱 한 번 보는 최종 시험입니다.

### OOF (Out-Of-Fold) 예측 — 알아두면 좋은 개념

교차검증을 하면, **각 소는 자기가 검증셋이었던 그 회차에 딱 한 번 예측**을 받습니다. 이 예측들을 다 모으면 "dev 전체에 대한, 자기 학습에 안 쓰인 깨끗한 예측"이 됩니다. 이걸 **OOF 예측**이라 합니다. 혼동행렬을 그릴 때 이 OOF 예측을 쓰면 누수 없이 전체를 평가할 수 있습니다. (C6 코드에서 다시 만납니다.)

## 4. 교차검증의 종류 — 그냥 자르면 안 되는 이유

`KFold`(그냥 자르기)는 **두 가지 상황**에서 위험합니다. 그래서 변형들이 있습니다. 하나씩 봅시다.

### 문제 ① 불균형 → Stratified(계층) 분할

그냥 무작위로 5조각 내면, 운 나쁘게 **등외(0.23%)가 어떤 fold엔 0마리** 일 수 있습니다. 그러면 그 fold에선 등외를 평가조차 못 합니다.

**해결: StratifiedKFold(계층적 분할)** — **각 fold에 등급 비율을 똑같이** 맞춰 나눕니다. 전체에 1++B가 13%면 모든 fold에도 13%, 등외가 0.23%면 모든 fold에도 0.23%. "계층(strata)을 유지한다"는 뜻입니다.

```
그냥 KFold:        fold마다 등급 비율이 들쭉날쭉 (등외 0마리인 fold 생길 수 있음)
StratifiedKFold:   모든 fold가 전체와 같은 등급 비율 ✓
```

### 문제 ② 그룹 누수 → Group 분할

이게 우리 프로젝트의 **핵심**입니다. 천천히 봅시다.

같은 **농장**의 소들은 사료·물·관리·심지어 유전(같은 씨수소)이 비슷해서 **서로 닮았습니다.** 만약 무작위로 나눠서 **같은 농장의 소가 학습셋과 검증셋에 동시에** 들어가면 무슨 일이 생길까요?

모델이 "이 농장 소들은 대충 이렇더라"를 **학습셋에서 외워서**, 검증셋의 같은 농장 소를 쉽게 맞힙니다. 실력이 아니라 **"농장 외우기" 컨닝**입니다. 그래서 검증 점수가 부풀고, 정작 **진짜 test의 처음 보는 농장**에선 폭락합니다. 이걸 **그룹 누수(group leakage)** 라고 합니다.

**해결: GroupKFold(그룹 분할)** — **같은 농장은 통째로 같은 fold에** 넣습니다. 한 농장이 학습과 검증에 쪼개져 들어가지 못하게. 그러면 검증셋엔 항상 "학습에서 본 적 없는 농장"만 들어가서, 진짜 일반화 성능을 잽니다.

> 비유: 시험 범위를 나눌 때 "같은 단원 문제는 같은 시험에" 몰아야지, 한 단원을 연습문제와 본시험에 쪼개면 본시험이 쉬워 보입니다.

### 두 문제를 동시에 → StratifiedGroupKFold (우리가 쓰는 것)

우리는 **불균형도 심하고(①) 농장도 닮았습니다(②).** 그래서 둘을 동시에 푸는 **StratifiedGroupKFold**를 씁니다.

- **Group**: 같은 농장(FARM_UNIQUE_NO)은 같은 fold로 → 그룹 누수 차단
- **Stratified**: 그러면서도 fold마다 등급 비율 유지 → 불균형 대응

이 분할은 이미 만들어져 `split_assignment.csv`에 저장돼 있고, 검증까지 끝났습니다(농장이 두 fold에 걸친 경우 0건, 모든 fold에 16등급 존재). **두 팀이 이 파일을 공유하고, 절대 새로 만들지 않습니다.**

### 정리 표

| 분할 방식 | 막는 문제 | 우리에게 |
| --- | --- | --- |
| KFold | (없음 — 그냥 자르기) | ✗ 위험 |
| StratifiedKFold | 불균형 | 부분적 |
| GroupKFold | 그룹 누수 | 부분적 |
| **StratifiedGroupKFold** | **불균형 + 그룹 누수** | **✓ 우리가 사용** |

## 5. 데이터 누수 총정리 — 4가지 얼굴

[00-B](00B_우리프로젝트_전체그림.md)에서 "누수 = 컨닝"이라고 배웠습니다. 누수는 여러 모습으로 나타납니다. **전부 외워두면 평생 써먹습니다.**

### ① 타깃 누수 (정답이 입력에 섞임)

가장 직접적. 타깃(또는 타깃에서 파생된 값)이 피처에 들어간 경우. 우리 예: **육질 측정값·WGRADE·가격** — 등급과 같은 시점에 결정돼 정답을 거의 담고 있습니다. → 분류 피처에서 제외.

### ② 전처리 누수 (검증셋 정보가 학습에 샘)

미묘하지만 흔합니다. 예를 들어 결측치를 **"전체 데이터의 중앙값"** 으로 채우면? 그 중앙값엔 **검증셋·test의 정보가 들어있습니다.** 학습이 검증셋을 미리 엿본 셈이죠.

**올바른 방법**: 중앙값·표준화 기준 등 모든 "데이터로부터 계산하는 값"은 **학습셋에서만** 계산하고, 검증셋·test엔 그걸 **적용만** 합니다. (우리 코드의 `prepare_X`가 이 원칙을 지킵니다. C6에서 봅니다.)

### ③ 그룹 누수 (위 4-②)

같은 농장이 학습·검증에 쪼개져 들어가 "외우기"가 가능해지는 것. → GroupKFold로 차단.

### ④ 시간 누수 (미래 정보로 과거 예측)

"미래에야 알 수 있는 값"으로 과거를 예측하는 것. 우리는 생애 전체 기간으로 집계하므로 직접적 시간 누수는 적지만, 만약 "도축 후" 정보를 쓰면 일종의 시간 누수입니다(육질 누수가 여기 겹칩니다).

> **누수 자가진단 한 문장**: *"내가 진짜 예측해야 하는 순간에, 이 값을 실제로 알 수 있나?"* 답이 "아니오"면 누수입니다. test에 그 컬럼이 있는지 보면 됩니다.

## 6. 불균형 다루기 — 클래스 가중치

[C1](C1_분류문제와_평가지표.md)에서 "Macro-F1은 소수 등급도 중요하게 본다"고 했습니다. 그런데 모델은 그냥 두면 **다수 등급에 유리하게** 학습합니다(틀려도 다수에서 맞히는 게 손실을 더 줄이니까). 이걸 바로잡는 가장 기본 도구가 **클래스 가중치**입니다.

### 아이디어: 소수 클래스의 실수에 더 큰 벌점

모델이 등외(소수)를 틀리면 **더 크게 혼냅니다.** 그러면 모델이 등외도 신경 쓰게 됩니다. "틀리면 벌금"인데, 희귀한 등급일수록 벌금을 비싸게 매기는 거죠.

### 공식

```
클래스 k의 가중치  wₖ = N / (K × nₖ)

  N  = 전체 표본 수 (240만)
  K  = 클래스 수 (16)
  nₖ = 클래스 k의 표본 수
```

예: 등외(5,540마리)의 가중치 ≈ 2,408,699 / (16 × 5,540) ≈ **27배.** 흔한 1++B는 약 0.57배. 즉 등외 한 마리를 틀리는 것이 1++B 약 47마리를 틀리는 것과 맞먹는 벌점이 됩니다. 이게 `class_weight="balanced"`가 자동으로 하는 일입니다.

### 다른 방법들 (넓게 알아두기)

클래스 가중치 말고도 불균형을 다루는 방법이 있습니다. 우리는 주로 가중치를 쓰지만 개념은 알아둡시다.

| 방법 | 아이디어 | 주의 |
| --- | --- | --- |
| **클래스 가중치** | 소수 클래스 오류에 큰 벌점 | 우리 기본. 데이터를 안 바꿔서 안전 |
| 오버샘플링 | 소수 클래스를 **복제**해 수를 늘림 | 똑같은 복제는 과적합 위험 |
| 언더샘플링 | 다수 클래스를 **버려서** 수를 줄임 | 정보 손실 |
| SMOTE | 소수 클래스 사이를 **보간해 가짜 샘플** 생성 | 남용 시 비현실적 샘플, 누수 주의 |

> 초보 단계에선 **클래스 가중치**가 가장 안전하고 효과적입니다. SMOTE 등은 "반드시 학습셋 안에서만, fold 분할 후에" 적용해야 누수가 안 생깁니다(분할 전에 하면 가짜 샘플이 검증셋에 새어 들어감).

## 7. 우리 분할을 한눈에

```
step11 (240만) 
   │  StratifiedGroupKFold (농장 그룹 + 등급 계층)
   ├─ holdout 20% (48만) ── 최종 1회 채점용
   └─ dev 80% (192만)
        │  다시 StratifiedGroupKFold (5-fold)
        └─ fold 0~4 ── 교차검증으로 학습·튜닝

진짜 test (45만, 정답 없음) ── 제출용, 위와 완전 분리
```

- 모든 분할은 **농장 단위**(그룹 누수 차단) + **등급 비율 유지**(불균형 대응).
- `split_assignment.csv`에 누가 dev/holdout이고 어느 fold인지 저장됨. **공유·고정.**

## 8. 핵심 정리

- 데이터를 나누는 유일한 이유: **외웠는지 vs 이해했는지**를 구분하려고(일반화 측정).
- 3분할: **학습**(공부) / **검증**(튜닝하며 여러 번) / **테스트**(끝나고 한 번).
- **교차검증(K-Fold)** = 조각을 돌려가며 K번 채점 → 데이터 낭비 없고, 평균·표준편차로 안정성까지 본다.
- **StratifiedKFold**(등급 비율 유지) + **GroupKFold**(농장 누수 차단) = **StratifiedGroupKFold**(우리).
- **누수 4종**: 타깃(정답 섞임)·전처리(검증 정보 샘)·그룹(농장 외우기)·시간(미래로 과거 예측). 자가진단: *"예측 순간 이 값을 진짜 아는가?"*
- 불균형 대응 기본 = **클래스 가중치**(소수 오류에 큰 벌점, wₖ = N/(K·nₖ)).
