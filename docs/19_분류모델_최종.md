### 1-2. 트랙1 규칙 — 누수 한 줄 요약

> 쓸 수 있는 변수의 기준은 "도축 전/후"가 아니라 **"test 13개 컬럼으로 계산 가능한가"** 다.
> WEIGHT·AGE·daily_gain은 가능(트랙1 OK), 육질 8개·WGRADE·COST_AMT는 불가(제출 모델 금지).

"참고-사후" 모델(육질 포함)은 **제출 불가**지만 "측정값이 있으면 얼마나 맞출 수 있는가"의 상한선을 보여주는 **보고서용 비교 분석**으로만 1회 돌린다.

### 1-3. 데이터가 3개다 — train / holdout / 진짜 test (혼동 주의)

가장 많이 헷갈리는 부분이다. 우리가 다루는 데이터는 **세 종류**다.

| | 무엇 | 정답(LAST_GRADE) | 행 수 | 역할 |
| --- | --- | --- | --- | --- |
| **train (정답 보유 전체)** | step11_features.csv | **있음** | 2,408,699 | 학습·검증의 재료 |
| ↳ **dev** | train의 80% | 있음 | 약 192만 | 5-fold CV로 학습·튜닝 |
| ↳ **holdout** | train의 20% | 있음 | 약 48만 | "처음 보는 데이터" 흉내 — 자체 최종 점검 1회 |
| **진짜 test (제출용)** | `data/test/test_hanwoo.csv` | **없음(전부 빈칸)** | 452,497 | 대회 제출 — 우리가 예측해서 채워 넣을 대상 |

핵심 구분:
- **holdout**은 train을 쪼갠 것이라 정답이 있어서 "내 모델이 몇 점인지" 우리가 직접 채점할 수 있다. 그래서 튜닝이 끝난 뒤 **단 한 번** 보는 자체 시험지다.
- **진짜 test**는 정답이 없다(대회 주최측만 안다). 우리는 예측값을 만들어 제출하고, 점수는 리더보드로만 안다. 그래서 holdout이 "test의 대역(代役)" 역할을 한다 — holdout 점수가 곧 제출 점수의 예상치다.

> **즉 "train/test 나눴냐?"에 대한 답**: ① 학습 검증용으로 train 안에서 dev/holdout을 농장 단위로 나눴고(이미 완료), ② 진짜 제출용 test는 별도 파일로 존재하며 **본 강의안 12장에서 동일 파이프라인으로 예측해 제출 파일을 만든다.** test에는 기상 파생이 없으므로(원본 13컬럼뿐) train과 똑같은 병합·전처리·파생을 그대로 적용하는 것이 12장의 핵심이다.

## 2. 이론 — 꼭 알고 시작할 것들

### 2-1. 정확도(Accuracy)의 함정

```
정확도 = 맞춘 수 / 전체 수
```

우리 데이터는 1++B 혼자 13.3%다. 16개 등급 중 상위 몇 개만 찍어도 정확도는 그럴듯하게 나온다. **불균형 58배에서 정확도는 거의 무의미**하다.

### 2-2. 정밀도·재현율·F1 (공식)

어떤 한 등급(예: 1++A)에 대해:

```
정밀도(Precision) = TP / (TP + FP)   — 모델이 "1++A"라고 한 것 중 진짜 비율
재현율(Recall)    = TP / (TP + FN)   — 진짜 1++A 중 모델이 잡아낸 비율
F1 = 2 · (정밀도 × 재현율) / (정밀도 + 재현율)   — 둘의 조화평균
```

조화평균이라서 **둘 중 하나라도 0에 가까우면 F1도 0에 가깝다** — 정밀도만 높고 재현율이 바닥인 "꼼수"를 막는다.

### 2-3. Macro-F1 (대회 평가지표)

```
Macro-F1 = (1/K) · Σₖ F1ₖ     (K = 16개 등급)
```

각 등급의 F1을 **표본 수와 무관하게 단순 평균**한다. 등외(5,540마리)의 F1과 1++B(319,588마리)의 F1이 **같은 가중치**다. → 소수 등급을 포기하는 모델은 Macro-F1이 크게 깎인다. 이것이 아래 모든 모델에 클래스 가중치를 넣는 이유다.

### 2-4. 클래스 가중치 (class_weight="balanced")의 공식

```
wₖ = N / (K · nₖ)     (N=전체 표본, K=클래스 수, nₖ=클래스 k의 표본 수)
```

등외처럼 드문 클래스의 오류에 더 큰 벌점을 줘서, 모델이 소수 클래스를 포기하지 않게 만든다. 예: 등외 w ≈ 2,408,699/(16×5,540) ≈ 27배 가중.

### 2-5. 농장 단위 분할 — 왜 일반 KFold면 안 되나

같은 농장의 소들은 사료·관리·유전이 비슷해서 서로 닮았다. 무작위로 나누면 **같은 농장의 소가 학습/검증 양쪽에 들어가**, 모델이 "농장 외우기"로 점수를 부풀린다(그룹 누수). 실제 test의 새 농장에서는 성능이 폭락한다.

→ **StratifiedGroupKFold**: 같은 농장(FARM_UNIQUE_NO)은 반드시 같은 fold에(Group), 등급 비율은 fold마다 유지(Stratified).

> 분할은 이미 만들어져 검증까지 끝났다(`split_assignment.csv`): 농장 누수 0건, 5개 fold 모두 16등급 포함, 홀드아웃 정확히 20%. **회귀팀과 같은 파일을 쓴다** — 두 트랙 결과를 비교 가능하게 만드는 장치다.

### 2-6. 그래디언트 부스팅 3형제 — 공통 원리와 차이

셋 다 **약한 트리를 순차적으로 쌓아 이전 모델의 오류(그래디언트)를 보정**하는 부스팅이다:

```
F₀(x) = 초기값
Fₘ(x) = Fₘ₋₁(x) + ν · hₘ(x)
   hₘ = 현재 모델의 손실 그래디언트(≈잔차)를 가장 잘 맞추는 트리
   ν  = learning_rate (한 걸음의 크기)
```

| | LightGBM | XGBoost | CatBoost |
| --- | --- | --- | --- |
| 트리 성장 | **leaf-wise** (손실 가장 큰 잎부터 분할 → 빠르고 깊게) | level-wise 기본 (층별 균등 분할 → 안정적) | **대칭 트리** (모든 층이 같은 분할 → 과적합에 강함) |
| 속도 | 가장 빠름 (히스토그램 기반) | 빠름 (`tree_method="hist"`) | 상대적으로 느림 |
| 강점 | 대용량·기본 성능 | 정밀한 정규화 (목적함수에 Ω(h)=γT+½λ‖w‖² 포함) | 순서형 부스팅으로 타깃 누수에 강함, 범주형 자동 처리 |
| NaN | 자동 처리 | 자동 처리 | 자동 처리 |
| 클래스 가중 | `class_weight="balanced"` | `sample_weight` 직접 전달 | `auto_class_weights="Balanced"` |

**로지스틱 회귀**는 비교 기준점(베이스라인)이다. 다항(소프트맥스) 형태:

```
P(y=k | x) = exp(βₖᵀx) / Σⱼ exp(βⱼᵀx)
```

선형 결정경계만 그릴 수 있고, 스케일에 민감해서 **표준화가 필수**이며, NaN을 못 받아 **중앙값 대치가 필수**다. "트리 모델이 이 단순한 모델보다 얼마나 나은가"를 재는 자(尺)다.

### 셀 3 (코드) — 데이터 로드 + 분할 병합

```python
# ==============================================================
# step11 로드 + 분할(split_assignment) 병합
# ==============================================================
df = pd.read_csv("../../data/processed/3_eda/step11_features.csv",
                 encoding="utf-8-sig", low_memory=False)
print(f"데이터: {df.shape}")   # (2408699, 92)

SPLIT_PATH = "../../data/processed/4_model/split_assignment.csv"

if os.path.exists(SPLIT_PATH):
    split = pd.read_csv(SPLIT_PATH, encoding="utf-8-sig")
    print("기존 분할 로드 (회귀팀과 공유) — 새로 만들지 말 것!")
else:
    # 최초 1회만 실행됨 — 농장 단위 + 등급 비율 유지 분할
    from sklearn.model_selection import StratifiedGroupKFold
    sgkf = StratifiedGroupKFold(n_splits=5, shuffle=True, random_state=SEED)
    dev_idx, hold_idx = next(sgkf.split(df, df["LAST_GRADE"],
                                        groups=df["FARM_UNIQUE_NO"]))
    df["split"] = "dev"; df.loc[hold_idx, "split"] = "holdout"
    df["fold"] = -1
    dev_tmp = df[df["split"] == "dev"]
    sgkf2 = StratifiedGroupKFold(n_splits=5, shuffle=True, random_state=SEED)
    for k, (_, va) in enumerate(sgkf2.split(dev_tmp, dev_tmp["LAST_GRADE"],
                                            groups=dev_tmp["FARM_UNIQUE_NO"])):
        df.loc[dev_tmp.index[va], "fold"] = k
    df[["CATTLE_NO","split","fold"]].to_csv(SPLIT_PATH, index=False,
                                            encoding="utf-8-sig")
    split = df[["CATTLE_NO","split","fold"]]
    print("분할 새로 생성 후 저장")

df = df.merge(split, on="CATTLE_NO", how="left") if "split" not in df.columns else df
print(df["split"].value_counts())
print("dev fold 분포:\n", df[df['split']=='dev']['fold'].value_counts().sort_index())

# 분할 자체 검증 — 농장이 두 영역에 걸치면 누수
farm_check = df.groupby("FARM_UNIQUE_NO")["split"].nunique()
assert (farm_check > 1).sum() == 0, "농장 누수 발견! 분할 파일 확인 필요"
print("농장 누수 검증 통과")
```

### 셀 6 (코드) — 개발/홀드아웃 분리 + 방어적 결측 처리 함수

```python
# ==============================================================
# dev/holdout 분리 + 결측 처리 유틸
# ==============================================================
dev  = df[df["split"] == "dev"].reset_index(drop=True)
hold = df[df["split"] == "holdout"].reset_index(drop=True)
print(f"개발: {len(dev):,} / 홀드아웃: {len(hold):,}")

def prepare_X(train_X, *other_Xs):
    """train 중앙값으로 결측 대치 (누수 방지: 중앙값은 train에서만 계산).
    ① inf→NaN ② train 중앙값 대치 ③ 그래도 NaN인(전부결측) 컬럼은 제거."""
    train_X = train_X.replace([np.inf, -np.inf], np.nan)
    med = train_X.median(numeric_only=True)
    out = [train_X.fillna(med)]
    for X_ in other_Xs:
        out.append(X_.replace([np.inf, -np.inf], np.nan).fillna(med))
    # 전부 결측이라 중앙값조차 없는 컬럼 방어 (실데이터 스모크에서 실제 발생)
    bad = out[0].columns[out[0].isnull().any()].tolist()
    if bad:
        print(f"  [방어] 전부 결측 컬럼 제거: {bad}")
        out = [o.drop(columns=bad) for o in out]
    return out if len(out) > 1 else out[0]

# fold 인덱스 리스트 (위치 기반)
fold_indices = [(np.where(dev["fold"] != k)[0], np.where(dev["fold"] == k)[0])
                for k in range(5)]
y_dev = dev["y"]

# ★ heat_high 임계값을 dev 분위 하나로 통일 (train·test 동일 기준선)
#   step11의 heat_high는 전체(2.4M) 분위로 만들어졌음 → dev 분위로 다시 계산해 일관성 확보
HEAT_Q75 = float(np.nanquantile(dev["ratio_고온"], 0.75))   # ← test에서도 이 값을 재사용
for part in (dev, hold):
    part["heat_high"] = (part["ratio_고온"] >= HEAT_Q75).astype(int)
X_t1 = dev[features_track1]                                  # heat_high 갱신 반영
print(f"heat_high 임계값(dev 75분위) = {HEAT_Q75:.4f}")
```

**중앙값을 train에서만 계산하는 이유**: 검증 fold의 중앙값을 쓰면 검증 정보가 학습에 새어 들어간다(전처리 누수). 트리 3종은 NaN을 자동 처리하므로 사실 대치가 불필요하지만, 로지스틱과의 **공정 비교**를 위해 같은 입력을 쓴다.

**heat_high를 다시 계산하는 이유**: 17회차에서 step11을 만들 때 heat_high 임계값을 전체 240만 행 기준으로 잡았다. 그런데 test에는 그 임계값이 없으니 dev에서 다시 구해야 하는데, 그러면 train(전체 분위)과 test(dev 분위)가 **서로 다른 기준선**을 쓰게 된다. 그래서 여기서 train(dev·hold)도 dev 분위로 다시 계산해 **train과 test가 똑같은 `HEAT_Q75`를 공유**하게 만든다. (80% 분위 vs 100% 분위라 값 차이는 작지만, 기준을 일치시키는 게 원칙.)

### 셀 7 (코드) — 공통 CV 러너

```python
# ==============================================================
# 공통 5-fold CV 러너 — 4개 모델을 같은 조건에서
# ==============================================================
def run_cv(name, make_model, X_all, y_all, fold_indices,
           n_folds=5, needs_scaling=False, use_sample_weight=False,
           fit_kwargs_fn=None):
    """make_model(): 새 모델 객체 반환. fit_kwargs_fn(X_va,y_va): fit 추가 인자."""
    results, oof = [], np.full(len(X_all), -1, dtype=int)
    last_model = None
    for fold_i, (tr_idx, va_idx) in enumerate(fold_indices[:n_folds]):
        X_tr, X_va = X_all.iloc[tr_idx], X_all.iloc[va_idx]
        y_tr, y_va = y_all.iloc[tr_idx], y_all.iloc[va_idx]
        X_tr, X_va = prepare_X(X_tr, X_va)

        if needs_scaling:
            sc = StandardScaler()
            X_tr = pd.DataFrame(sc.fit_transform(X_tr), columns=X_tr.columns)
            X_va = pd.DataFrame(sc.transform(X_va),  columns=X_va.columns)

        model = make_model()
        kwargs = fit_kwargs_fn(X_va, y_va) if fit_kwargs_fn else {}
        if use_sample_weight:   # XGBoost용 — 클래스 균형 가중치
            kwargs["sample_weight"] = compute_sample_weight("balanced", y_tr)
        model.fit(X_tr, y_tr, **kwargs)

        pred = np.asarray(model.predict(X_va)).ravel().astype(int)  # ★CatBoost (n,1) 대응
        oof[va_idx] = pred
        f1m = f1_score(y_va, pred, average="macro")
        acc = accuracy_score(y_va, pred)
        results.append({"fold": fold_i, "macro_f1": round(f1m,4), "acc": round(acc,4)})
        print(f"  [{name}] fold {fold_i}: Macro-F1={f1m:.4f}, Acc={acc:.4f}")
        last_model = model
    r = pd.DataFrame(results)
    print(f"  [{name}] 평균 Macro-F1 = {r['macro_f1'].mean():.4f} ± {r['macro_f1'].std():.4f}")
    return r, oof, last_model
```

`.ravel().astype(int)` 한 줄이 CatBoost의 2차원 predict를 흡수한다 — 4개 모델이 한 함수로 돈다.

```python

N_TRIALS = 20           # TODO: 시간에 따라 조정 (10~50). 1 trial ≈ 수 분

        n_estimators=trial.suggest_int("n_estimators", 300, 1500, step=100),

        colsample_bytree=trial.suggest_float("colsample_bytree", 0.6, 1.0),
        reg_alpha=trial.suggest_float("reg_alpha", 1e-3, 10.0, log=True),
        reg_lambda=trial.suggest_float("reg_lambda", 1e-3, 10.0, log=True),
```

**탐색 공간 설명** — 각 파라미터가 무엇을 조절하나:
| 파라미터 | 늘리면 | 과적합 위험 |
| --- | --- | --- |
| `learning_rate` ↓ + `n_estimators` ↑ | 더 정밀한 학습 | ↓ (대신 느려짐) |
| `max_depth`, `num_leaves` ↑ | 더 복잡한 패턴 포착 | ↑ |
| `min_child_samples` ↑ | 잎이 더 일반적 | ↓ |
| `subsample`, `colsample_bytree` ↓ | 무작위성 증가 | ↓ |
| `reg_alpha`(L1), `reg_lambda`(L2) ↑ | 가중치 억제 | ↓ |

## 3.5 대회 test 예측 & 제출 파일 만들기 (★ 실제 제출의 핵심)

지금까지는 정답이 있는 train 안에서만 놀았다. 이제 **정답이 없는 진짜 test 452,497마리의 등급을 예측해서 제출 파일을 만든다.** test는 원본 13컬럼뿐이라, **train에 했던 병합·전처리·파생을 토씨 하나 안 틀리게** 다시 적용해야 한다. 한 군데라도 다르면 모델이 엉뚱한 숫자를 받아 예측이 무너진다.

### 누수 방지 3대 원칙 (test 처리에서 반드시)

1. **빈도·통계는 train에서 배운 것을 test에 "적용"만** 한다 (test로 다시 학습 금지). 혈통 빈도 맵, 결측 대치 중앙값이 여기 해당.
2. **더미 컬럼은 train 목록에 맞춰 정렬**한다. test에만 있는 카테고리는 버리고, train에 있는데 test에 없으면 0으로 채운다 (모델 입력 컬럼 순서·개수가 정확히 같아야 함).
3. **기상 집계는 같은 날씨 테이블**(step4-3)로 train과 동일 방식. test 관측소 199곳은 전부 날씨 테이블에 있음을 확인했다.

### 셀 20 (코드) — test 피처 빌드 함수 (검증 완료)

```python
# ==============================================================
# test_hanwoo.csv → step11과 동일 구조의 피처로 변환
#   (실데이터로 병합·기상집계·파생·빈도·더미 전 구간 검증된 코드)
# ==============================================================
def build_features_for_test(test_path="../../data/test/test_hanwoo.csv"):
    test = pd.read_csv(test_path, low_memory=False)
    for c in ["ABATT_DATE", "BIRTH_YMD", "JUDGE_DATE"]:
        test[c] = pd.to_datetime(test[c], errors="coerce")

    # ── (1) 농장/개체 병합: death, area, lineage (train과 동일 규칙) ──
    death = pd.read_csv("../../data/raw/hanwoo_death.csv")
    test = test.merge(death.groupby("FARM_UNIQUE_NO").size()
                      .rename("death_count").reset_index(),
                      on="FARM_UNIQUE_NO", how="left")
    area = pd.read_csv("../../data/raw/hanwoo_area.csv")
    for c in ["C2023", "C2024", "C2025", "AREA"]:
        area[c] = area[c].replace(-99, np.nan)
    test = test.merge(area.groupby("FARM_UNIQUE_NO")[["C2023","C2024","C2025","AREA"]]
                      .sum().reset_index(), on="FARM_UNIQUE_NO", how="left")
    lin = pd.read_csv("../../data/raw/hanwoo_lineage.csv", low_memory=False)
    test = test.merge(lin, on="CATTLE_NO", how="left")

    # ── (2) 기상 생애 노출 집계 (step4-4 누적합 방식, 동일 날씨 테이블) ──
    w = pd.read_csv("../../data/processed/1_merge/step4-3_weather.csv",
                    usecols=["stn","date","THI_grade","rn_day","ws_davg","ta_min"],
                    low_memory=False)
    w["date"] = pd.to_datetime(w["date"])
    for g in ["양호","주의","경고","위험","폐사"]:
        w[f"d_{g}"] = (w["THI_grade"] == g).astype(int)
    sum_cols = ["d_양호","d_주의","d_경고","d_위험","d_폐사","rn_day","ws_davg","ta_min"]
    test = test.reset_index(drop=True)
    n = len(test)
    days_arr = np.full(n, np.nan); seg = np.full((n, len(sum_cols)), np.nan)
    for stn, ws in w.groupby("stn"):
        ws = ws.sort_values("date"); dates = ws["date"].values
        csum = np.vstack([np.zeros(len(sum_cols)), ws[sum_cols].cumsum().values])
        idx = np.where(test["stn"].values == stn)[0]
        if len(idx) == 0:
            continue
        lo = np.searchsorted(dates, test["BIRTH_YMD"].values[idx], side="left")
        hi = np.searchsorted(dates, test["ABATT_DATE"].values[idx], side="right")
        seg[idx] = csum[hi] - csum[lo]; days_arr[idx] = hi - lo
    test["days_total"] = days_arr
    for i, g in enumerate(["양호","주의","경고","위험","폐사"]):
        test[f"days_{g}"] = seg[:, i]
    with np.errstate(invalid="ignore", divide="ignore"):
        test["rn_day_mean"]  = seg[:, 5] / days_arr
        test["ws_davg_mean"] = seg[:, 6] / days_arr
        test["ta_min_mean"]  = seg[:, 7] / days_arr

    # ── (3) 기상 비율 (15회차) ──
    test["ratio_고온"]   = (test["days_주의"]+test["days_경고"]+test["days_위험"]) / test["days_total"]
    test["ratio_강더위"] = (test["days_경고"]+test["days_위험"]) / test["days_total"]
    test["ratio_위험"]   = test["days_위험"] / test["days_total"]

    # ── (4) 시간·효율·임계·농장·상호작용 파생 (16~17회차) ──
    test["abatt_year"]    = test["ABATT_DATE"].dt.year
    test["abatt_month"]   = test["ABATT_DATE"].dt.month
    test["abatt_quarter"] = test["ABATT_DATE"].dt.quarter
    test["birth_year"]    = test["BIRTH_YMD"].dt.year
    test["birth_month"]   = test["BIRTH_YMD"].dt.month
    def to_season(m):
        return ("봄" if m in [3,4,5] else "여름" if m in [6,7,8]
                else "가을" if m in [9,10,11] else "겨울")
    test["abatt_season"] = test["abatt_month"].map(to_season)
    test["birth_season"] = test["birth_month"].map(to_season)
    test["fattening_days"] = (test["ABATT_DATE"] - test["BIRTH_YMD"]).dt.days
    test["daily_gain"]     = test["WEIGHT"] / test["fattening_days"]
    test["Age_squared"]    = test["AGE"] ** 2
    test["age_optimal"]    = ((test["AGE"] >= 28) & (test["AGE"] <= 32)).astype(int)
    # heat_high: 셀 6에서 dev로 구한 HEAT_Q75를 그대로 사용 (train과 동일 기준선, test 재계산 금지)
    test["heat_high"] = (test["ratio_고온"] >= HEAT_Q75).astype(int)
    test["density"]    = (test["C2025"] / test["AREA"]).replace([np.inf,-np.inf], np.nan)
    test["death_rate"] = (test["death_count"] / test["C2025"]).replace([np.inf,-np.inf], np.nan)
    test["density_x_heat"] = test["density"] * test["ratio_고온"]

    # ── (5) 혈통 빈도 인코딩: TRAIN에서 배운 빈도를 매핑 (누수 방지) ──
    for col in ["KPN_NO","FATHER_CATTLE_NO","MOTHER_ANIMAL_NO"]:
        freq_map = (df[[col, f"{col}_freq"]].dropna()
                      .drop_duplicates().set_index(col)[f"{col}_freq"].to_dict())
        test[f"{col}_freq"] = test[col].map(freq_map)   # train에 없는 신규값 → NaN

    # ── (6) 원-핫 더미: train 컬럼에 맞춰 정렬 ──
    test["sex"] = test["JUDGE_SEX"]   # get_dummies prefix 맞추기용
    dummies = pd.get_dummies(test[["sido","abatt_season","birth_season","sex"]],
                             prefix=["sido","abatt_season","birth_season","sex"]).astype(int)
    train_dummy_cols = [c for c in features_track1
                        if c.startswith(("sido_","abatt_season_","birth_season_","sex_"))]
    for col in train_dummy_cols:
        test[col] = dummies[col] if col in dummies.columns else 0   # 없으면 0
    return test

test_feat = build_features_for_test()
print(f"test 피처화 완료: {test_feat.shape}")

# 트랙1 피처가 빠짐없이 만들어졌는지 검증
missing_cols = [c for c in features_track1 if c not in test_feat.columns]
assert not missing_cols, f"누락 피처: {missing_cols}"
print(f"트랙1 {len(features_track1)}개 피처 전부 생성 확인")
```

## 4. 결과 해석 가이드 — 숫자를 받아들고 무엇을 볼 것인가

> 결과 숫자는 여러분이 직접 돌려서 얻는다. 아래는 "읽는 순서"다.

**① CV 평균 ± 표준편차부터.** 평균이 성능, 표준편차가 안정성이다. fold 간 편차가 크면(예: ±0.02 이상) 모델이 데이터 구획에 민감하다는 뜻 — 파라미터보다 피처/분할을 먼저 의심한다.

**② Macro-F1과 정확도의 격차.** 정확도는 높은데 Macro-F1이 낮으면 다수 등급만 잘 맞추고 소수 등급을 버리고 있다는 신호다. classification_report에서 어느 등급의 recall이 바닥인지 확인하라.

**③ 혼동행렬은 대각선 주변을 본다.** 한우 등급은 순서가 있어서 **인접 등급 혼동(1++B↔1++A)은 가벼운 오류, 건너뛰기 혼동(1++A↔2C)은 심각한 오류**다. 행 기준 % 행렬에서 오류가 대각선 부근에 몰려 있으면 모델이 "품질의 순서"는 학습한 것이다. 특히 같은 숫자 등급 내 A/B/C 혼동(예: 1+A vs 1+B — 육량 차이)이 많은지, 숫자 등급 간 혼동(1+ vs 1 — 근내지방 차이)이 많은지를 구분해 보라. 전자가 많다면 육량 관련 피처(WEIGHT, daily_gain)가, 후자가 많다면 품질 관련 신호가 부족한 것이다.

**④ 트랙1 vs 참고-사후의 격차가 보고서의 핵심 문장.** 사후(육질 포함)가 크게 높게 나오는 것이 정상이다 — 등급은 육질 측정값으로 매겨지기 때문. 그 격차 자체가 "사전 정보만으로 예측의 한계는 어디까지인가"라는 분석 결과다.

**⑤ SHAP 중요도는 EDA와 맞춰본다.** 13~17회차에서 daily_gain(r=+0.522), WEIGHT(+0.475), 성별이 강했다. SHAP 상위권이 이와 일치하면 모델이 합리적으로 배우고 있는 것이고, 전혀 뜬금없는 변수가 1위면 누수나 버그를 의심하라. summary plot에서는 점 색(변수값 높낮이)과 좌우 위치(SHAP 부호)를 함께 읽는다 — "daily_gain이 높을수록(빨강) 1++A 확률을 올린다(오른쪽)" 같은 문장으로.

**⑥ CV vs 홀드아웃 격차 = 일반화 점검.** 거의 같으면 분할·모델링이 건강한 것. 홀드아웃이 눈에 띄게 낮으면 (농장 분할 덕에 가능성은 낮지만) CV에 과적합한 것이다.

**⑦ 등외 등급의 F1은 따로 언급하라.** 5,540마리(0.23%)뿐이고 노폐우 등 이질 집단이라 F1이 낮게 나올 수 있다. 보고서에 "등외 제외 시 Macro-F1" 도 병기하면 분석의 정직성이 드러난다.

## 5. 성능 향상 가이드 — 일주일 동안 시도할 것들 (우선순위순)

### 5-1. 파라미터 손대기 (효과 큰 순)

| 우선순위 | 무엇을 | 어떻게 | 기대 효과 |
| --- | --- | --- | --- |
| 1 | `learning_rate` ↓ + `n_estimators` ↑ | 0.02~0.03 + 2000~3000 (early stopping 믿고) | 거의 항상 소폭 상승. 시간↑ |
| 2 | `num_leaves` / `max_depth` | Optuna 범위를 num_leaves 31~511로 확장 | 데이터가 커서 복잡도 여유 있음 |
| 3 | `min_child_samples` | 20~300 탐색 | 소수 등급 과적합/과소적합 균형 |
| 4 | `N_TRIALS` ↑ | 50~100회 (밤새 돌리기) | 탐색 공간 충분히 커버 |
| 5 | Optuna 평가를 다중 fold로 | objective에서 fold 0,1 평균 사용 | 튜닝 결과의 신뢰성 ↑ (시간 2배) |

### 5-2. 클래스 가중 커스텀 (Macro-F1 직공)

"balanced"는 출발점일 뿐이다. Macro-F1은 모든 클래스를 동등하게 보므로, **소수 클래스 가중을 balanced보다 더 키우거나 줄이며** 최적점을 찾아라:

```python
# 예: balanced 가중에 지수 γ를 적용 (γ=1이 balanced, γ>1이면 소수 클래스 더 강조)
counts = y_dev.value_counts().sort_index().values
w = (len(y_dev) / (n_classes * counts)) ** GAMMA      # GAMMA를 0.5~1.5 탐색
class_weight = {i: w[i] for i in range(n_classes)}
# LGBMClassifier(class_weight=class_weight, ...)
```

### 5-3. 피처 추가 — 시군구 OOF Target Encoding (18회차에 미뤘던 것)

sigungu(184종)는 정보가 많지만 그대로 못 쓴다. **fold 안에서만** 평균 타깃으로 인코딩하면 누수 없이 쓸 수 있다:

```python
def add_te(dev, col="sigungu", target="grade_num"):
    te = np.full(len(dev), np.nan)
    for k in range(5):
        tr_mask = dev["fold"] != k
        mapping = dev[tr_mask].groupby(col)[target].mean()      # fold 밖 데이터만으로
        te[~tr_mask.values] = dev.loc[~tr_mask, col].map(mapping).values
    dev[f"{col}_te"] = te
    return dev
# 홀드아웃/test에는 dev 전체로 만든 mapping을 적용
```

같은 방식으로 `stn`(관측소), `eupmyeondong`도 시도 가치가 있다. **주의**: mapping은 반드시 해당 fold의 train에서만 — 전체로 만들면 그게 바로 Target Encoding 누수다.

### 5-4. 모델 구조 아이디어

- **앙상블(soft voting)**: LGB·XGB·CatBoost의 `predict_proba` 평균 후 argmax. 모델들이 서로 다른 실수를 하면 1~3%p 오르는 경우가 흔하다.
- **2단계 분해**: 등급 = 숫자등급(1++/1+/1/2/3/등외, 6클래스) × 육량(A/B/C, 3클래스). 두 모델로 나눠 예측 후 조합 — 클래스당 표본이 늘어 소수 등급에 유리할 수 있다.
- **순서형(ordinal) 활용**: 등급에 순서가 있으므로 grade_num 회귀 후 반올림→클래스 변환도 비교해 볼 것 (Macro-F1로 동일 평가).
- **등외 별도 처리**: 등외(노폐우 다수)만 먼저 이진 분류로 걸러내고 나머지 15등급을 분류하는 2단계.

## 7. 총정리

1. **문제**: 16등급, 58배 불균형 → 평가지표는 Macro-F1, 모든 모델에 클래스 가중 필수.
2. **누수 차단 2중 장치**: 피처는 "test에서 계산 가능한 것"만(EXCLUDE 블랙리스트), 분할은 농장 단위(StratifiedGroupKFold, 회귀팀과 공유 파일).
3. **모델 사다리**: 로지스틱(기준점) → LightGBM(메인) → XGBoost/CatBoost(교차 확인) → Optuna(튜닝) → 앙상블(여력 시).
4. **해석**: 혼동행렬은 "인접 vs 건너뛰기" 오류로, SHAP은 EDA 결과(daily_gain·WEIGHT·성별)와 대조해서 읽는다.
5. **규율**: 튜닝은 CV 안에서 끝내고 홀드아웃은 한 번. 참고-사후(육질) 모델은 보고서 비교용일 뿐 절대 제출하지 않는다.
6. **버전 함정 3종** (CatBoost 2차원 predict, shap 3차원 배열, xgboost 생성자 early stopping)은 본 강의안 코드에 이미 반영돼 있다.
