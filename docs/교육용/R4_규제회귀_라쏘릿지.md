# R4. 규제 회귀 — Lasso·Ridge·ElasticNet

> **이 강의의 목표**: 변수가 많거나 닮았을 때 계수를 "억눌러서" 모델을 단순·안정적으로 만드는 **규제(regularization)** 를 배웁니다. L1·L2 규제가 무엇이고, Lasso·Ridge·ElasticNet이 각각 어떻게 다른지, 그리고 이게 [R3](R3_다중공선성_VIF.md)의 다중공선성을 어떻게 푸는지.
> **앞 강의**: [R2](R2_선형회귀_기초.md)의 OLS·최소제곱, [R3](R3_다중공선성_VIF.md)의 다중공선성, [C5](C5_튜닝과_모델해석.md)의 과적합.

## 1. OLS의 한계 — 변수가 많으면 흔들린다

[R2](R2_선형회귀_기초.md)의 OLS는 "잔차 제곱 합을 최소화"만 합니다. 변수가 적고 깔끔하면 좋은데, 변수가 **많거나 서로 닮으면**([R3](R3_다중공선성_VIF.md)) 문제가 생깁니다.

- 계수가 **불안정**해지고(데이터 조금 바뀌면 출렁),
- 일부 계수가 **비정상적으로 커지며**(한쪽이 +100, 다른쪽이 −100처럼 상쇄),
- 학습 데이터에 **과적합**([C5](C5_튜닝과_모델해석.md))하기 쉽습니다.

이걸 푸는 아이디어가 **규제**입니다.

---

## 2. 규제란 — "계수를 너무 키우지 마"라는 벌금

규제(regularization)의 핵심 아이디어는 단순합니다.

> OLS는 "잔차만 줄여라"였는데, 규제 회귀는 **"잔차도 줄이되, 계수도 너무 크게 만들지 마"** 라고 조건을 하나 더 겁니다.

```
OLS:        잔차 제곱 합을 최소화
규제 회귀:  (잔차 제곱 합) + (계수 크기에 대한 벌금)  을 최소화
                                    ↑ 이게 규제 항
```

계수가 커지면 벌금이 늘어나니, 모델이 **꼭 필요한 만큼만 계수를 키우게** 됩니다. 결과적으로 더 단순하고 안정적인 모델이 됩니다.

### 비유: 짐 쌀 때 무게 제한

여행 가방을 쌀 때 "필요한 건 다 넣어"라고만 하면 과하게 챙깁니다. "**무게 제한이 있으니 정말 필요한 것만**"이라고 하면 우선순위를 따져 알짜만 넣죠. 규제가 이 무게 제한입니다 — 모델이 "정말 중요한 변수에만 큰 계수를" 쓰게 만듭니다.

### 규제 강도 α (알파)

벌금을 얼마나 세게 매길지가 **α(alpha)** 입니다.

- α가 **크면** → 벌금이 세서 계수가 많이 눌림 → 모델 단순(과소적합 쪽).
- α가 **작으면** → 벌금이 약해서 OLS에 가까움 → 모델 복잡(과적합 쪽).

적절한 α를 찾는 게 규제 회귀의 튜닝입니다([C5](C5_튜닝과_모델해석.md)의 하이퍼파라미터처럼).

---

## 3. 두 종류의 벌금: L1과 L2

"계수 크기에 벌금"을 매기는 방식이 두 가지입니다. 이 차이가 Lasso와 Ridge를 가릅니다.

### L2 규제 (제곱 벌금) → Ridge

계수를 **제곱해서** 더한 걸 벌금으로:

```
벌금 = α × (b₁² + b₂² + b₃² + ...)
```

제곱이라 **큰 계수에 특히 큰 벌금**이 갑니다. 그래서 큰 계수를 집중적으로 눌러 **전체를 고르게 작게** 만듭니다. 단, **계수를 정확히 0으로 만들지는 않습니다**(작게 줄일 뿐).

### L1 규제 (절댓값 벌금) → Lasso

계수의 **절댓값**을 더한 걸 벌금으로:

```
벌금 = α × (|b₁| + |b₂| + |b₃| + ...)
```

절댓값 벌금은 수학적 성질 때문에 **일부 계수를 정확히 0으로** 만듭니다. 즉 **쓸모없는 변수를 아예 빼버립니다.** 이게 Lasso의 마법 — **자동 변수 선택**입니다.

> 왜 L1은 0을 만들고 L2는 못 만드나? 절댓값 함수는 0에서 **뾰족한 모서리**가 있어서, 최적해가 그 모서리(=0)에 딱 걸리기 쉽습니다. 제곱 함수는 0에서 **부드러워서** 0 근처까진 가도 정확히 0엔 안 닿습니다. (직관만 기억하면 됩니다.)

---

## 4. 세 모델 정리

### Ridge (L2) — 고르게 줄이기

모든 변수를 살려두되 계수를 전체적으로 **압축**. 다중공선성([R3](R3_다중공선성_VIF.md))을 완화하는 데 특히 좋습니다 — 닮은 변수들의 계수가 한쪽으로 쏠리지 않고 고르게 나뉘니까요. "변수를 버리긴 싫고 안정만 시키고 싶을 때."

### Lasso (L1) — 변수 골라내기

쓸모없는 변수의 계수를 **0으로** 만들어 **자동으로 솎아냅니다.** 변수가 수십 개일 때 "어떤 게 진짜 중요한가"를 알려줍니다. 단, **닮은 변수 그룹에선 하나만 남기고 나머지를 다 버리는** 경향이 있어 가끔 과합니다.

### ElasticNet (L1+L2) — 둘의 절충

Lasso와 Ridge를 **섞습니다.** `l1_ratio`로 비율 조절(0이면 순수 Ridge, 1이면 순수 Lasso, 0.5면 반반). Lasso의 "닮은 변수 그룹에서 하나만 남기는" 단점을 Ridge가 보완해줍니다. "변수 선택도 하고 싶고 안정성도 원할 때."

### 비교 표 (핵심)

| 모델 | 벌금 | 계수를 0으로? | 강점 | 언제 |
| --- | --- | --- | --- | --- |
| **Ridge** | L2(제곱) | ✗ (작게만) | 다중공선성 완화 | 변수 다 살리고 안정화 |
| **Lasso** | L1(절댓값) | ✓ (변수 선택) | 자동 변수 선택 | 중요 변수만 추리기 |
| **ElasticNet** | L1+L2 | ✓ (덜 공격적) | 둘의 균형 | 선택+안정 둘 다 |

## 5. 규제 회귀의 두 가지 필수 주의

### 주의 ① 표준화는 필수다

규제는 **계수 크기**에 벌금을 매깁니다. 그런데 [C3](C3_인코딩과_결측_스케일링.md)에서 봤듯 변수마다 단위가 다르면 계수 크기도 단위 때문에 달라집니다(몸무게 계수와 더위 비율 계수는 크기 자체가 비교 불가). 표준화를 안 하면 **단위가 큰 변수가 부당하게 더 큰 벌금**을 받습니다. 그래서 **규제 회귀 전엔 반드시 표준화**(학습셋 기준!).

### 주의 ② 규제 계수로는 "해석"하지 않는다

이게 중요합니다. 규제는 계수를 **일부러 눌러서(편향시켜서)** 안정성을 얻습니다. 즉 규제 계수는 "진짜 효과"보다 **줄어든 값**입니다. 그래서 **"더위 효과가 N%"** 같은 정량 해석은 **OLS로** 하고([R5](R5_계수해석_한계효과.md)), 규제 모델은 **변수 선택과 예측 안정성 확인**에만 씁니다.

```
OLS:        계수 해석 (더위 효과의 크기·신뢰구간) ← 보고서 숫자
규제 회귀:  어떤 변수가 살아남나(Lasso), 예측이 안정적인가 ← 보조 증거
```

> 우리 트랙2 전략: **OLS가 해석의 본체**, **규제 3종은 "그 결론이 모델을 바꿔도 견고한가"의 증인.** 네 모델의 예측력이 비슷하고 Lasso에서도 더위 변수가 살아남으면, "결과가 모델 선택에 둔감하다"는 강건성 증거가 됩니다.

## 6. 규제와 다중공선성 — R3와의 연결

[R3](R3_다중공선성_VIF.md) 끝에서 "미묘한 다중공선성은 규제에 맡긴다"고 했죠. 이제 이해됩니다.

- 닮은 변수 둘이 OLS에선 +100/−100처럼 폭주하는데,
- **Ridge**는 둘의 계수를 고르게 눌러 안정시키고,
- **Lasso**는 둘 중 하나를 0으로 만들어 정리합니다.

그래서 VIF로 **명백한 중복(월령↔사육일수)만 손으로 제거**하고, 나머지 미묘한 닮음은 규제가 알아서 처리하는 게 효율적입니다.

---

## 7. 핵심 정리

- **규제** = "잔차도 줄이되 계수도 키우지 마"라는 벌금 → 모델을 단순·안정·과적합 방지.
- **α(알파)** = 벌금 강도. 크면 단순(과소적합), 작으면 OLS에 가까움(과적합). 튜닝 대상.
- **L2(Ridge)** = 제곱 벌금, 계수를 **고르게 축소**(0은 안 됨), 다중공선성 완화.
- **L1(Lasso)** = 절댓값 벌금, 일부 계수를 **0으로**(자동 변수 선택).
- **ElasticNet** = L1+L2 절충(`l1_ratio`).
- 규제 회귀는 **표준화 필수**(학습셋 기준), **계수 해석은 OLS로**(규제 계수는 편향됨).
- 규제는 [R3](R3_다중공선성_VIF.md) 다중공선성도 완화 → 명백한 중복만 손으로 빼고 나머진 규제에 맡김.
