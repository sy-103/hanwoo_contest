# C6. 실습 — 코드 따라 하기

> **이 강의의 목표**: C1~C5에서 배운 개념이 **실제 코드에서 어떻게 구현되는지** 봅니다. 코드 전체는 실습 강의안 [`19_분류모델_최종.md`](../19_분류모델_최종.md)에 셀별로 다 있으니, 여기서는 **각 셀이 어떤 개념을 구현하는지** 지도를 그리고, 처음 돌릴 때 막히는 부분을 짚습니다. 코드를 외울 필요는 없습니다 — "이 셀이 왜 있는지"만 이해하면 됩니다.


> 🗺️ **학습 여정**: 기초(00A·00B) → 데이터준비(D1·D2) → EDA(E1·E2·E3) → 〔분류 C1–C7〕 · 〔회귀 R1–R8〕  ·  **📍 지금: 분류 6/7**

## 1. 실습 준비 — 환경 만들기

코드를 돌리기 전에 도구를 깔아야 합니다. 터미널(또는 노트북 첫 셀)에서:

```bash
pip install pandas numpy matplotlib seaborn scikit-learn lightgbm xgboost catboost optuna shap joblib

# ★ Mac 사용자 필수! LightGBM이 이걸 요구합니다 (없으면 import부터 에러)
brew install libomp
```

> **왜 libomp?** LightGBM은 내부적으로 병렬 계산(OpenMP)을 쓰는데, Mac엔 그 라이브러리가 기본으로 없습니다. 안 깔면 `import lightgbm`에서 `dlopen ... libomp.dylib` 에러가 납니다. 처음 겪으면 한참 헤매는 함정이라 미리 알려둡니다.

확인용으로 버전을 찍어보고, 하나라도 에러면 위 설치부터 다시 합니다.

## 3. 꼭 이해하고 넘어갈 셀 5개

전부 중요하지만, **개념과 직결돼 꼭 짚어야 할** 다섯 개만 깊이 봅니다.

### 셀 4 — 피처 고르기: "넣을 것"이 아니라 "뺄 것"을 적는다

```python
EXCLUDE = {
    "LAST_GRADE", "grade_num",            # 타깃 자신
    "BACKFAT","REA",...,"COST_AMT",       # 육질·가격 = 도축 후 = test에 없음(누수)
    "CATTLE_NO","FARM_UNIQUE_NO",...,      # ID·해시 (의미 없음)
    "ABATT_DATE",...,                      # 날짜 원본(파생으로 대체)
    "sido","sigungu",...,                  # 글자 원본(더미로 대체)
    "days_폐사",                            # 상수(전부 0)
}
features_track1 = [숫자형 컬럼 중 EXCLUDE에 없는 것]
```

**왜 이렇게?** [C2](C2_데이터분할과_누수.md)에서 배운 **타깃 누수**를 막는 핵심입니다. "넣을 것"을 나열하면 17회차에서 만든 파생변수를 빠뜨리기 쉽지만, **"뺄 것을 이유와 함께"** 적으면 모든 변수를 검토했다는 게 코드로 증명됩니다. 특히 육질·가격을 빼는 게 생명입니다(넣으면 점수는 오르지만 컨닝).

### 셀 6 — 결측 처리 함수: 전처리 누수 방지의 실체

```python
def prepare_X(train_X, *other_Xs):
    med = train_X.median(...)        # ★ 중앙값을 train에서만 계산
    out = [train_X.fillna(med)]
    for X_ in other_Xs:
        out.append(X_.fillna(med))   # 검증/test엔 train 중앙값을 "적용만"
    ...
```

이 작은 함수가 [C2](C2_데이터분할과_누수.md)·[C3](C3_인코딩과_결측_스케일링.md)의 **전처리 누수 방지**를 구현합니다. 중앙값을 검증셋 포함 전체로 구하면 누수, **학습셋에서만** 구해 나머지에 적용하면 안전. 트리는 NaN을 그냥 먹지만, 로지스틱과 공정 비교를 위해 같은 입력을 씁니다.

> 같은 셀의 `heat_high` 재계산도 같은 원리 — 임계값(75분위)을 dev에서 구해 train·test에 똑같이 적용해 기준선을 통일합니다.

### 셀 7 — 교차검증 러너: 4모델을 같은 잣대로

```python
def run_cv(name, make_model, X, y, fold_indices, ...):
    for fold_i, (tr_idx, va_idx) in enumerate(fold_indices):
        ...학습...
        pred = np.asarray(model.predict(X_va)).ravel().astype(int)  # ★ ravel
        f1m = f1_score(y_va, pred, average="macro")                 # ★ Macro-F1
```

[C2](C2_데이터분할과_누수.md)의 5-fold를 구현. 4개 모델을 **똑같은 fold·똑같은 지표**로 돌려 공정 비교합니다. `.ravel()`은 CatBoost가 예측을 `(n,1)` 2차원으로 주는 버릇을 펴주는 것(안 하면 점수가 이상해짐 — 찾기 어려운 버그).

### 셀 9~11 — 세 부스팅, 클래스 가중치 거는 법이 다름

[C4](C4_분류모델_4종.md)의 3형제. **불균형 대응([C2](C2_데이터분할과_누수.md) 클래스 가중치)을 거는 방법이 모델마다 다릅니다.**

```python
LightGBM:  class_weight="balanced"            # 인자로
XGBoost:   sample_weight=compute_sample_weight("balanced", y)  # fit에 직접
CatBoost:  auto_class_weights="Balanced"      # 인자로
```

같은 "소수 등급에 큰 벌점"인데 거는 문이 다를 뿐입니다. 모든 모델에 가중치를 넣는 이유는 [C1](C1_분류문제와_평가지표.md)의 Macro-F1 때문 — 소수 등급을 포기하면 점수가 깎이니까.

### 셀 18 — 홀드아웃: 딱 한 번의 규율

```python
holdout_model = lgb.LGBMClassifier(**best_params)
holdout_model.fit(X_dev_full, y_dev)         # dev로만 학습
pred_hold = holdout_model.predict(X_hold)    # holdout으로 1회 채점
```

[C2](C2_데이터분할과_누수.md)의 홀드아웃 규율. **이 점수를 보고 모델을 또 고치면 안 됩니다.** holdout으로 학습하지 않고(그러면 자기시험), dev로 학습한 모델을 holdout에 1회만 적용해 "처음 보는 데이터 점수"를 얻습니다.

## 4. 처음 돌릴 때 자주 만나는 에러

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| `import lightgbm`에서 dlopen libomp 에러 | Mac에 OpenMP 없음 | `brew install libomp` |
| 그림의 한글이 □□로 깨짐 | 폰트 미설정/순서 문제 | 셀 2의 폰트 설정(sns.set_style을 폰트보다 먼저) |
| CatBoost 점수가 이상하게 낮음 | predict가 2차원인데 ravel 안 함 | `.ravel()` 확인 |
| XGBoost `early_stopping_rounds` 에러 | 2.x는 fit이 아니라 생성자 인자 | 생성자에 넣기 |
| 메모리 부족/너무 느림 | 240만 행 전체 학습 | 로지스틱은 샘플, 트리는 FAST 플래그(fold 0만) |
| 분할 파일 없음 | split_assignment.csv 미생성 | 셀 3이 없으면 만들어 주지만, 있으면 로드만 |

> 처음엔 **FAST 모드**(XGB·CatBoost를 fold 0만 돌리는 플래그)로 끝까지 한 번 완주하세요. 전체 5-fold는 시간이 오래 걸리니, 흐름을 익힌 뒤 켭니다.

## 5. 실습 진행 순서 (추천)

1. **환경 설치** + 버전 확인 셀 통과.
2. 셀 2~7까지 돌려 **데이터·분할·함수** 준비. (여기서 농장 누수 assert가 통과하는지 꼭 확인)
3. 셀 8~12로 **4모델 베이스라인** 비교. 어느 모델이 좋은지 감 잡기.
4. 셀 14~15 **Optuna 튜닝**(처음엔 N_TRIALS=10~20으로 빠르게).
5. 셀 16~17 **혼동행렬·SHAP**으로 모델이 어떻게·왜 예측하는지 확인.
6. 셀 18 **홀드아웃 1회**.
7. 셀 19~21 **저장·제출** ([C7](C7_제출과_성능향상.md)).
